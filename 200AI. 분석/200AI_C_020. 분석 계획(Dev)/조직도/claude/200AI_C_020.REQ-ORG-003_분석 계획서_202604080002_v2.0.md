### GAP 대응 방안

1. **SaaS 멀티 도메인 가시성 설정 활용**: Smarts4j의 `sys_object_domain` 테이블 및 `DomainID=0`(그룹사) 기반 통합 조직도 트리 구성 기능을 활용한다. `selectInitOrgTreeList` 쿼리에 도메인 필터 확장을 적용하여 `AllDomain=Y` 파라미터 시 V1·V2 통합 트리를 반환한다.
2. **isSaaS 파라미터 적용**: SYNC FieldMapping.sql의 `insertTempObjectDept`, `insertTempObjectUser`, `insertTempObjectRegion` 등에서 `isSaaS=Y` 설정 시 GroupCode에 `CompanyCode_` 접두어를 붙여 V1·V2 간 코드 충돌을 방지한다. (FieldMapping.sql의 QUERY 타입 MemberOf/GroupCode 필드에서 `isSaaS` 파라미터로 처리됨을 확인.)
3. **전자결재 조직도 팝업 확장**: 결재선/참조 선택 팝업 호출 시 `isDisplayAllDomain` 파라미터를 추가하여 V1·V2 통합 조직도를 표시한다. 단, 결재선 권한(ACL)은 기존 도메인 정책 유지.
4. **이음톡 연동 API 도메인 필터 확장**: 이음톡에서 조직도 조회 시 양사 데이터가 포함되도록 API 파라미터 확장.
5. **계열사 간 정보 노출 범위 협의**: 타 계열사 사용자 정보(전화번호, 메일, 사진 등) 노출 범위를 고객과 협의하여 `sys_object_domain` 설정 또는 별도 ACL 정책으로 반영.

### 3-2. 기술적 타당성

#### 구현 난이도

| 항목 | 난이도 | 근거 |
|------|--------|------|
| `selectInitOrgTreeList` 멀티 도메인 확장 | 중 | 기존 동적 쿼리 구조에 도메인 조건 추가. SQL 최적화 필요 |
| isSaaS 파라미터 적용 (FieldMapping.sql) | 중 | 현재 FieldMapping.sql에서 QUERY 타입 sourceField 내 `#{_parameter[isSaaS]}` 사용 중. JDBC 파라미터 바인딩 이슈(SQLSyntaxErrorException) 해결 필요 → sourceField 문자열 내 `#{_parameter[isSaaS]}` 를 Java 단에서 실제값으로 치환하는 전처리 로직 추가 필요 |
| 전자결재 조직도 팝업 확장 | 중 | 결재선 팝업 호출 파라미터 추가, 통합 조직도 표시 |
| 이음톡 연동 API 확장 | 중 | 이음톡 조직도 API의 도메인 필터 확장 |
| ACL 정책 (타 도메인 가시성) | 고 | 계열사 간 보안 정책 결정이 선행되어야 함. 정책 미확정 시 개발 범위 불명확 |
| 대용량 조직도 성능 | 중 | V1+V2 통합 조직도 로드 시 레벨별 지연 로딩(Lazy Loading) 최적화 필요 |

#### 제약 사항

1. **isSaaS 파라미터 JDBC 이슈**: 현재 FieldMapping.sql의 QUERY 타입 sourceField 내에 MyBatis `#{_parameter[isSaaS]}` 표현식이 포함되어 있어, `INSERT INTO ... SELECT` 구문에서 MariaDB Connector/J의 다중 파라미터 바인딩 오류(`Could not set parameter at position N`) 발생 가능. Java 서비스 단에서 사전 치환 로직 추가 필요.
2. **계열사 간 정보 노출 범위 미확정**: 고객사 협의가 완료되어야 ACL 정책 구현 가능. 미확정 시 '전체 공개' 기본값으로 개발 후 정책 적용 시 수정.
3. **성능 저하 가능성**: V1+V2 통합 조직도 트리 로드 시 데이터량 증가로 인한 성능 저하 가능. Lazy Loading 및 Redis 캐시 전략 수립 필수.
4. **양사 GroupCode 충돌**: `isSaaS=Y` 미적용 시 V1·V2 동일한 GroupCode 충돌 발생 가능. FieldMapping.sql 적용 전 검증 필요.

#### 대안 제시

- **ACL 정책 미확정 시**: 단계적 구현 전략 적용. 1단계: 양사 전체 조직도 통합 표시(읽기 권한). 2단계: 고객 협의 후 민감 정보(전화번호, 메일) 필드별 노출 정책 적용.
- **성능 최적화**: 조직도 트리 최초 로드 시 레벨 1~2만 반환하고, 하위 레벨은 사용자 확장 시 비동기 로드(Lazy Loading). Redis 캐시 TTL 설정으로 반복 조회 성능 확보.
- **isSaaS 파라미터 이슈 해결**: `OrgSyncService` 또는 `OrgSyncCon`에서 FieldMapping 목록을 조회한 후 QUERY 타입 sourceField 내 `#{_parameter[isSaaS]}` 패턴을 실제 isSaaS 값(`'Y'` 또는 `'N'`)으로 치환하는 전처리 메서드를 추가한다.
