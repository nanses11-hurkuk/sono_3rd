#### DB 변경점

소스 코드의 SQL 매핑 파일 분석 결과를 기반으로 다음 DB 변경점을 확인하였습니다.

- `xform_xformEditor.xml`의 `selectFormList` 쿼리에서 `covi_approval4j.jwf_forms` 테이블, `covi_approval4j.jwf_formsSchema` 테이블, `covi_approval4j.jwf_formclass` 테이블을 조인하여 양식 목록을 조회하고 있습니다. 15종 양식 메타데이터(`FormID`, `FormName`, `FormPrefix`, `CompanyCode`, `Revision` 등) 등록이 필요합니다.
- `xform_xformEditor.xml`의 `jwf_XFormTemplate` 테이블에서 양식 템플릿을 관리하고 있으므로, 유사 양식 간 공통 템플릿 활용 가능성을 검토해야 합니다.
- `admin_adminForm.xml`의 `selectAdminFormList` 쿼리에서 `jwf_forms_acl` 테이블로 양식별 접근 권한(ACL)을 관리하고 있으므로, 15종 양식 각각의 접근 권한 설정이 필요합니다.

#### 인프라 변경점

- `XFormEditorSvc.java`의 `getHTMLFormFile()`, `getJSFormFile()` 메서드에서 양식 파일을 `formCompanyCode` 기반으로 조회하고 있으므로, 양식 렌더링 시 네트워크 지연을 고려한 정적 리소스 캐싱 전략이 필요합니다.
- TECHNICAL_GUIDE_COMMON_SERVICES.md의 파일 관리 서비스(`fileDown.do`)의 `bizSection` 설정에 신규 양식 관련 저장 경로를 추가해야 합니다.

**산출 근거**: 소스코드(`smarts4j.coviapproval/src`) xform/form/admin 패키지 분석, 분석 요청서(200AI_C_010.REQ-WFW-002) 영향도 분석 항목, TECHNICAL_GUIDE_COMMON_SERVICES.md 파일 관리 서비스 bizSection 기반 격리 구조 참조

### 3-2. 기술적 타당성

#### 구현 난이도

- 소스 분석 결과, `XFormEditorSvc.java`에 `createHTMLFormFile()`, `createJSFormFile()`, `setTemplate()` 등 양식 생성/관리 표준 인터페이스가 이미 구현되어 있습니다.
- `XFormEditorCon.java`에서 Jsoup 라이브러리를 사용하여 HTML 양식을 파싱/편집하는 구조이므로, xFormEditor를 통한 양식 개발은 표준화된 도구를 활용할 수 있습니다.
- 개별 양식당 구현 난이도는 **낮은** 수준이나, 15종 양식의 규격이 미확정 상태이므로 양식 확정 후 실제 공수가 변동될 수 있습니다.

#### 제약 사항

- 양식 종류 및 규격이 미확정 상태(15종)이므로, 고객사와의 양식 확정 일정이 전체 개발 일정에 직접적인 영향을 미칩니다.
- `admin_adminForm.xml`에서 양식 검색 시 `FormPrefix`, `FormClassName`, `FormName` 기준으로 필터링하고 있으므로, 15종 양식의 명명 규칙(FormPrefix) 사전 정의가 필요합니다.

#### 대안 제시

- `xform_xformEditor.xml`의 `jwf_XFormTemplate` 테이블을 활용하여 공통 양식 템플릿 구조를 먼저 설계하고, 양식 확정 후 `createHTMLFormFile()`/`createJSFormFile()`로 개별 양식을 빠르게 적용하는 2단계 접근을 권장합니다.
- xFormEditor의 템플릿 복제(`insertTemplateData`) 기능을 활용하여 유사 양식 간 개발 효율을 높일 수 있습니다.

**산출 근거**: 소스코드(`XFormEditorSvc.java`, `XFormEditorCon.java`, `xform_xformEditor.xml`) 분석 결과, 분석 요청서(200AI_C_010.REQ-WFW-002) 기술적 타당성 항목 기준

### 3-5. API 및 서비스 인터페이스 영향도

소스 코드에서 확인된 API 인터페이스 영향 범위는 다음과 같습니다.

- **양식 CRUD API** (`XFormEditorCon.java`): `selectFormList()`, `createHTMLFormFile()`, `createJSFormFile()`, `setTemplate()` 인터페이스를 통해 15종 양식 등록/조회/수정/삭제가 수행됩니다.
- **양식 관리 Admin API** (`AdminFormCon.java`): 관리자 화면에서 양식 메타데이터 관리(`selectAdminFormList`) 및 양식-ACL 매핑 인터페이스를 검토해야 합니다.
- **양식 조회 API** (`FormCon.java` / `FormSvc.java`): `selectForm()`, `selectFormSchema()`, `getBodyData()` 등 양식 데이터 조회 인터페이스와의 연동을 확인해야 합니다.
- **파일 관리 서비스** (`fileDown.do`): 양식 첨부파일 처리 인터페이스와의 연동을 확인해야 합니다.
- **통합 메시징 서비스** (`sendMessage.do`): 양식 결재 알림 발송 인터페이스를 검토해야 합니다.

### 3-6. 데이터 정합성 및 트랜잭션 전략

- `xform_xformEditor.xml`에서 `jwf_forms`, `jwf_formsSchema`, `jwf_formclass` 테이블이 조인 관계에 있으므로, 양식 등록/수정 시 마스터-상세 간의 트랜잭션 원자성(Atomicity)을 보장해야 합니다.
