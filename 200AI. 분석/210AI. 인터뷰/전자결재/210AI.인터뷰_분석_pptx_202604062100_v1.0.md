## 1. 결재 프로세스 유형

### 1.1 보고/품의 유형

**정의**: 품의서, 보고서와 같이 별도의 주관부서가 필요 없는 경우의 결재 프로세스입니다.

**프로세스 흐름**: 기안자 → 결재자(순차) → 결재 프로세스 종료

**결재선 형태**: 기안자부터 최종결재자까지 순서대로 배치하며, 문서의 우측 상단에 결재 Box를 표시합니다. Sub Process의 합의자/협조자는 결재 Box와 결재문서 본문 중간에 Table 형태로 표시하고, 참조자는 Sub Process의 결재 Box와 결재문서 본문 중간에 Table 형태로 표시합니다.

#### 1.1.1 결재 용어

| 결재 용어 | 참고 용어 | 설명 | 고객사 용어 |
|---|---|---|---|
| 기안 | 작성 | 기안자가 결재문서를 작성하여 결재 요청하는 행위 | 기안 |
| 승인 | 결재 | 결재자가 승인 요청한 문서의 내용을 확인 후 승인하는 행위 | 승인 |
| 반려 | 기각 | 결재자가 승인 요청한 문서의 내용을 확인 후 반려하는 행위 | 반려 |

**산출 근거**: 인터뷰 PPTX 슬라이드 7 - 결재 용어 확인 테이블

#### 1.1.2 Sub Process 사용 여부

| 결재 구분 | 결재 유형 | 설명 | 반려시 처리 | 고객사 사용여부 |
|---|---|---|---|---|
| 협조 | 개인협조 | 일반결재와 동일하게 협조가 반려했을 경우 반려 처리됨. 순차(순차적으로 협조자에게 문서 배달) 및 병렬(N개의 협조자에게 동시에 문서 전달) 모두 지원 | 반려 완료 | **O** (순차, 병렬 모두 사용) |
| 협조 | 부서협조 | 해당 부서 수신함에서 문서를 열어 재기안 혹은 결재를 함. 부서장이 담당자를 지정하고 담당자가 미결함에 수신된 문서를 재기안. 순차/병렬 지정 가능 | 반려 완료 | **O** |
| 합의 | 개인합의 | 협조자가 동의하지 않고 거부하더라도 문서는 진행됨. 순차(순차적으로 합의자에게 문서 배달) 및 병렬(N개의 합의자에게 동시에 문서 전달) 지원 | 결재 진행 | **X** |
| 합의 | 부서합의 | 해당 부서 수신함에서 문서를 열어 재기안 혹은 결재를 함. 부서장이 담당자 지정하고 담당자가 미결함에 수신된 문서를 재기안. 순차/병렬 지정 가능 | 결재 진행 | **X** |

> **참고**: 합의(개인합의, 부서합의)와 협조(개인협조, 부서협조)의 핵심 차이는 반려 시 처리 방식입니다. 협조는 반려 시 결재 프로세스가 종료되고, 합의는 반려(거부)하더라도 결재 프로세스가 계속 진행됩니다.

**산출 근거**: 인터뷰 PPTX 슬라이드 7 - Sub Process 사용 여부 확인 테이블

#### 1.1.3 참조/회람 사용여부

| 결재 용어 | 참조 유형 | 설명 | 고객사 사용여부 | 비고 |
|---|---|---|---|---|
| 참조 | 참조 | 결재 문서를 다른 사용자/부서가 볼 수 있도록 지정하는 행위 (결재선 지정 시) | **O** | |
| 참조 | 개인참조 | 사용자를 대상으로 참조 지정. 해당 사용자는 개인 참조회람함에서 문서 조회 가능 | **O** | |
| 참조 | 부서참조 | 부서를 대상으로 참조 지정. 소속 사용자는 부서 참조회람함에서 문서 조회 가능 | **O** | |
| 참조 | 사전참조 | 기안 시점에 조회 가능 (진행 중 문서 조회 가능) | **O** (개인/부서 모두 사용) | 옵션 |
| 참조 | 사후참조 | 결재완료(최종) 시점에 조회 가능 | **O** (개인/부서 모두 사용) | 기본 |
| 회람 | 회람 | 결재 완료된 문서를 다른 사용자/부서에서 볼 수 있도록 지정하는 행위. 지정된 사용자 또는 부서의 참조회람함에서 문서를 조회 | **O** (개인/부서 모두 사용) | |

> **참고**: Sub Process와 참조는 모든 결재 유형(보고/품의, 신청, 배포)에서 공통적으로 사용됩니다.

### 1.2 신청 유형

**정의**: 비용신청서, 구매신청과 같이 담당(주관) 부서의 승인을 필요로 하는 경우의 결재 프로세스입니다.

**프로세스 흐름**:
- **신청 유형 #1**: 기안부서 결재 → 처리부서(주관부서) 결재 → 결재 프로세스 종료
- **신청 유형 #2**: 기안부서 결재 → 1차 처리부서 결재 → 2차 처리부서 결재 → 결재 프로세스 종료 (담당 부서가 2개 이상인 경우 순차 처리)

**결재선 형태**:
- 신청 유형 #1: 기안부서의 결재자들을 좌측 상단에, 처리부서의 결재자들을 우측 상단에 결재 Box로 표시합니다.
- 신청 유형 #2: 기안부서를 좌측 상단에, 1차/2차 처리부서의 결재자들을 우측 상단에 순서대로 배치합니다.

#### 1.2.1 담당자, 담당부서, 담당업무함 사용여부

| 접수유형 | 지정주체 | 유형 | 설명 | 고객사 사용여부 |
|---|---|---|---|---|
| 담당자 | 기안자 | 개인 | 기안자가 주관부서 담당자를 결재선 지정 시 직접 지정함. 개인 미결함에 문서 도착. 해당 담당자는 본인 이후에 결재자를 추가할 수도 있고 본인만 결재할 수도 있음 | 검토 후 결정 |
| 담당자 | 관리자가 설정 | 개인 | 관리자가 양식프로세스에서 고정으로 한 담당자를 지정하므로, 담당자 성명이 결재선에 나타나며 기안자는 별도 지정할 필요 없음 | 검토 후 결정 |
| 담당부서 | 기안자 | 부서 | 기안자가 결재선 지정 시 담당부서를 직접 지정함. 해당 부서의 수신함에 문서 도착. 해당 부서의 부서원 누구나 결재선을 지정하여 재기안 | 검토 후 결정 |
| 담당부서 | 관리자가 설정 | 부서 | 관리자가 양식프로세스에서 고정으로 담당부서를 지정하므로, 담당부서가 결재선에 나타나며 기안자는 별도 지정할 필요 없이 자동으로 주관부서 수신함에 문서 도착 | 검토 후 결정 |
| 담당업무함 | 관리자가 설정 | 개인 | 관리자가 담당업무함을 별도로 생성시키고, 담당업무함별로 담당자를 멀티로 지정. 결재문서는 담당업무함의 미결함에 도착. 업무담당자가 다수인 경우 활용하며 인수인계가 수월. 기안자 결재선에는 해당업무명이 나타남 | 검토 후 결정 |

> **참고**: 담당자/담당부서/담당업무함 관련 항목은 모두 "검토 후 결정" 상태이므로, 고객사 확인이 필요합니다.

**산출 근거**: 인터뷰 PPTX 슬라이드 11 - 담당자, 담당부서, 담당업무함 사용여부 확인 테이블

### 1.3 배포 유형

**정의**: 협조요청, 지원요청과 같이 기안부서, 수신부서의 개별적 승인처리를 필요로 하는 경우의 결재 프로세스입니다. 수신부서의 의사결정에 따라 원문서가 반려되지 않는 다수 공지성 프로세스입니다.

**프로세스 흐름**: 기안부서 결재 → 수신처(개인/부서)에 배포 → 수신부서 개별 결재 → 결재 프로세스 종료

**결재선 형태**:
- 발신부서: 우측 상단에 결재자 Box 표시, 하단에 수신처(개인/부서) 표시
- 수신부서: 좌측 상단에 발신부서 결재자 Box 표시, 우측 상단에 수신부서 결재자 Box 표시

#### 1.3.1 수신처 유형

| 접수유형 | 유형 | 설명 | 고객사 사용여부 |
|---|---|---|---|
| 수신 | 수신자 | 수신문서를 개인이 수신함 (발신부서에서는 수신현황에서 결과 조회 가능). 수신자 미결함에 문서 도착 → 반송/접수/결재선을 지정하여 재기안 | **O** |
| 수신 | 수신부서 | 수신문서를 부서가 수신함 (발신부서에서는 수신현황에서 결과 조회 가능). 부서 수신함에 문서 도착 → 반송/접수/결재선을 지정하여 재기안. 해당 부서의 부서원 누구나가 결재선을 지정하여 재기안 | **O** |

**산출 근거**: 인터뷰 PPTX 슬라이드 14 - 수신처 유형 테이블

## 2. 결재 종류

| 결재행위 | 표시이름(예) | 설명 | 고객사 사용여부 |
|---|---|---|---|
| 결재 | 승인, 반려, 보류 | 결재자 의사결정 행위에 대한 명칭 | **O** (승인, 반려, 보류 사용) |
| 전결 | 전결 | 뒤에 결재자가 있더라도 전결로 지정되어 있으면 뒷 결재자는 결재안함으로 변경되고, 결재선에 전결로 글자가 표시됨. 회사의 문화에 따라 표기 여부 결정 | **O** |
| 후결 | 후결 | 최종 결재완료 후 결재하는 행위로, 후결자 지정 시 순서에 관계없이 맨 마지막에 결재하게 됨 (반려권한 없음, 후결자가 결재하기 전에는 문서가 종료되지 않음). 후결재는 미결함에서 확인만 가능 | **O** |
| 대결 | 대결 | 원결재자 대신 결재를 하는 행위로, 원결재자 및 대결자에게 동시에 미결함에 문서가 수신됨. 원결재자가 승인할 경우 대결자는 진행 상황만 조회, 대결자가 승인할 경우 결재란에는 대결로 표시됨 | **O** |

## 3. 결재 사용 기능

| 기능종류 | 설명 | 고객사 사용여부 |
|---|---|---|
| 서명표시 | 결재양식 화면에 개인의 서명 이미지가 보여지며 인쇄됨. 개인이 서명이미지를 등록/관리함 | **O** |
| 결재암호 | 결재 시 개인이 결재 암호를 넣어 본인임을 확인할 수 있게 해주는 기능. 암호는 DB상에 암호화되어 저장됨. 개인이 결재암호 사용여부를 결정함 | **O** |
| 알림 매체 | 미결함에 문서 도착 시 결재자들에게 결재도착 알림. 사용자가 개인정보 설정에서 사용여부 및 도착매체(메일, 모바일) 선택 | **O** |
| 알림 Event | 결재 Event 발생 시 알림. 사용자가 개인정보 설정에서 사용여부 및 도착매체(메일, 모바일) 선택. 결재 Event: 결재 요청, 완료, 반려, 회수, 취소, 보류, 참조, 수신, 담당업무, 회람 | **O** |
| 지정반려 | 현 결재자가 기 결재자 중에 한 명을 지정하여 결재문서를 반송하는 기능. 지정된 결재자부터 결재가 다시 진행됨. 일반결재자만 가능하고 합의/협조자는 불가 | **O** |
| 기안취소, 회수 | 기안자가 진행중인 결재 문서를 취소하는 기능. 기결재자에게 취소 사유를 메일로 발송함. 취소된 문서는 기안자 임시함에 보관됨. 회수는 결재자들이 한명도 결재하지 않은 경우 회수 가능 (기본기능) | **O** |
| 승인취소 | 차상위자가 대기중인 경우 이전 결재자의 미결함으로 문서를 회수하는 기능. 합의/협조자가 결재선에 있을 경우는 불가. 반려 또는 추가 의견 달기를 위해 사용 | **O** |
| 내용수정기능 | 결재자 결재 시 내용을 수정할 수 있는 기능. 전자결재 관리자가 양식마다 옵션 설정으로 결재자에게 수정버튼 활성화. 수정 History(이전문서/현재문서 조회) 관리됨 | **O** |
| 일괄확인/일괄결재/연속결재 | 미결함 목록에서 다중 선택된 결재문서를 결재하는 기능. 일괄결재: 리스트에서 선택된 문서를 동시에 승인 처리. 연속결재: 순차적으로 확인하면서 승인/반려 처리. 일괄확인: 리스트에서 선택한 문서를 순차적으로 확인. 양식마다 선택적으로 설정 가능 | **O** |
| 1인결재 | 결재선에서 기안자 1인으로 기안/완료처리를 수행함. 양식마다 선택적으로 허락할 수 있음 | **O** |
| 전달 | 결재문서의 결재자를 내가 아닌 다른 사람으로 교체. 전달 이력이 남지 않음 (관리툴에서만 조회 가능) | **X** |
| 기밀 | 결재문서 조회 권한 여부를 결정. 설정 시 부서완료함에 저장되지 않고 결재선에 포함된 사용자만 문서조회가 가능 | **O** |

**산출 근거**: 인터뷰 PPTX 슬라이드 16~17 - 결재 사용 기능 테이블

## 4. 결재 양식 표준 및 결재 공통 영역

### 4.1 결재 양식 표준 구성

결재 양식은 다음 영역으로 구성됩니다.

1. **결재 기능 버튼 표시** — 상단에 결재 관련 기능 버튼을 배치합니다.
2. **결재 양식명 표시** — 양식의 이름을 표시합니다.
3. **결재 Box 표시** — 결재선의 결재자 정보를 표시합니다.
4. **문서 공통 속성 표시** — 문서의 공통 속성 정보를 표시합니다.
5. **결재양식 본문 표시** — 결재문서의 본문 내용을 표시합니다.
6. **결재양식 하단 공통 영역** — 합의/협조, 수신처, 참조 등을 표시하며, 결재자 의견, 참고사항 등의 정보도 함께 표시합니다.

**산출 근거**: 인터뷰 PPTX 슬라이드 18 - 결재 양식 표준 레이아웃

### 4.2 결재 양식의 문서 속성

결재양식의 상단에 표시되는 주요 문서 속성으로 모든 결재 양식에 공통 적용됩니다.

| 문서 속성 | 설명 | 고객사 사용여부 |
|---|---|---|
| 작성자 | 문서를 작성하는 사용자 정보 | **O** |
| 작성부서 | 문서를 작성하는 사용자의 소속 부서 | **O** |
| 작성일자 | 문서의 작성일자로 기안자의 결재선에 작성일자를 표시 | **O** |
| 보존년한 | 문서의 보존년한 (1년/3년/5년/10년/영구). 문서의 속성 정보일 뿐이며 별도의 추가적인 의미는 없음 | **O** |
| 보안등급 | 문서의 보안 등급 (일반문서/대외비/비밀문서) | **O** |
| 문서분류 | 문서 분류. 필요시 문서관리 시스템으로 이관시키기 위한 용도로 사용될 수 있음 | **O** |
| 문서번호 | 문서 번호 (예: 연구1-2003-0001) | **O** |
| 제목 | 문서의 제목 | **O** |
| 기타 | 추가로 사용되는 문서 속성이 있는 경우 작성 | **?** (미확정) |

> **참고**: 작성자, 작성부서는 문서 속성 정보 부분에 주로 표시하고, 작성일자는 결재선의 기안일시 표시 부분에 주로 표시합니다. 제목처럼 필수 입력 Field는 별도로 구분 표시합니다.

**산출 근거**: 인터뷰 PPTX 슬라이드 19 - 결재 양식의 문서 속성 테이블

### 4.3 문서번호 정보

| 구분 | 설명 | 고객사 사용여부 |
|---|---|---|
| 문서번호 발번 규칙 | 문서번호를 생성하는 규칙. 주로 "부서명-YYYY-일련번호(4자리)" 형태를 많이 사용. 총 13종의 발번 형태 중 선택 가능하며 추가적인 발번 규칙이 필요한 경우 추가 가능 | **검토 후 결정** (기본적으로 부서명-YYYY-일련번호(4자리) 사용) |
| 문서번호 발번 시점 | 결재가 최종 완료되는 시점에 문서번호를 발번함. 기안 시점에 발번하는 경우 결재 반려시 결번이 생길 수 있음. 수신처의 경우 별도 발번 없이 발신부서 문서번호를 공유 | **결재 완료시 발번** |
| 문서번호 초기화 시점 | 년도가 바뀌는 경우 초기화 (0001부터 새롭게 시작). 년도가 바뀌는 기준은 결재 종료일자 기준 | **년도별 일련번호 부여** |

**산출 근거**: 인터뷰 PPTX 슬라이드 20 - 문서번호 정보 테이블

## 5. 결재선 Box 표시 형식

### 5.1 보고/품의 유형 또는 배포 유형(발신부서)의 결재선 Box 형태

- 기안자부터 최종결재자까지 순서대로 배치 (문서의 우측 상단에 결재 Box 표시)
- Sub Process의 합의자/협조자들을 표시 (결재 Box와 결재문서 본문 중간에 Table 형태로 표시)
- (사전/사후) 참조자들을 표시 (Sub Process 결재 Box와 결재문서 본문 중간에 Table 형태로 표시)
- 수신처가 있는 경우 표시 (수신자, 수신부서)

### 5.2 신청 유형 또는 배포 유형(수신부서)의 결재선 Box 형태

- 기안부서(또는 발신부서)의 결재자들을 순서대로 배치 (문서의 좌측 상단에 결재 Box 표시)
- 처리(또는 수신) 부서의 결재자들을 순서대로 배치 (문서의 우측 상단에 결재 Box 표시)
- Sub Process의 합의자/협조자들을 표시
- (사전/사후) 참조자들을 표시

### 5.3 결재 형태에 따른 결재선 Box 표시 규칙

| 결재 상태 | 표시 규칙 |
|---|---|
| 기안 | 기안자만 "기안"으로 표시하고, 그 외에는 직책을 표시 |
| 승인 | 서명 또는 인장이 등록된 경우 서명/인장 표시, 미등록인 경우 이름 표시. 결재한 사용자의 이름과 승인 여부, 결재 일시 표시 |
| 미결 | 공란으로 표시 |
| 대결 | 승인 글자 대신에 "대결"로 표시. 대결자의 인장이 표시됨 (원결재자를 결재현황 보기에서 확인 가능) |
| 후결/전결 | 승인 글자 대신에 결재유형("후결", "전결")을 표시 |
| 반려 | 결재자 인장이나 이름 대신에 "반려"로 표시 |

**산출 근거**: 인터뷰 PPTX 슬라이드 22~24 - 결재선 Box 표시 형식
