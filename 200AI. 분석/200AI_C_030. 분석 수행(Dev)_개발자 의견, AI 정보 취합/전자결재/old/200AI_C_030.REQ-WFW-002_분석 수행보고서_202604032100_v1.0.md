- `xform_xformEditor.xml`의 `selectFormList` 쿼리에서 `covi_approval4j.jwf_forms` 테이블에 양식 메타데이터(`FormID`, `FormName`, `FormPrefix`, `CompanyCode`, `Revision`, `SchemaID`, `FormClassID` 등) 등록이 필요합니다.
- `xform_xformEditor.xml`의 `jwf_XFormTemplate` 테이블에서 양식 템플릿을 관리하고 있으므로, 유사 양식 간 공통 템플릿을 활용하여 개발 효율을 높일 수 있습니다.
- `admin_adminForm.xml`의 `jwf_forms_acl` 테이블로 양식별 접근 권한(ACL)을 관리하고 있으므로, 15종 양식 각각의 접근 권한 설정이 필요합니다.

#### 인프라 변경점

- `XFormEditorSvc.java`의 `getHTMLFormFile()`, `getJSFormFile()` 메서드에서 양식 파일을 `formCompanyCode` 기반으로 조회하고 있으므로, 15종 양식 파일의 정적 리소스 캐싱 전략이 필요합니다.

### 3-2. 기술적 타당성

#### 구현 난이도

- 소스 분석 결과, `XFormEditorSvc.java`에 `createHTMLFormFile()`, `createJSFormFile()`, `setTemplate()` 등 양식 생성/관리 표준 인터페이스가 이미 구현되어 있습니다.
- `XFormEditorCon.java`에서 Jsoup 라이브러리를 사용하여 HTML 양식을 파싱/편집하는 구조이므로, xFormEditor를 통한 양식 개발은 표준화된 도구를 활용할 수 있습니다.
- 인터뷰 자료에서 확인된 문서 속성(작성자~제목 8종)이 공통 필드로 존재하므로, 공통 양식 템플릿에 이를 반영하여 15종 양식의 개발 효율을 높일 수 있습니다.
- 개별 양식당 구현 난이도는 **낮은** 수준이나, 15종 양식의 규격이 미확정 상태이므로 양식 확정 후 실제 공수가 변동될 수 있습니다.

#### 제약 사항

- 양식 종류 및 규격이 미확정 상태(15종)이므로, 고객사와의 양식 확정 일정이 전체 개발 일정에 직접적인 영향을 미칩니다.
- `admin_adminForm.xml`에서 양식 검색 시 `FormPrefix`, `FormClassName`, `FormName` 기준으로 필터링하고 있으므로, 15종 양식의 명명 규칙(FormPrefix) 사전 정의가 필요합니다.
- 인터뷰에서 문서번호 발번 규칙이 "부서명-YYYY-일련번호(4자리)"로 확인되었으나, 규칙 확정이 필요한 상태("검토 후 결정")이므로, 양식별 문서번호 체계 협의가 필요합니다.

#### 대안 제시

- `xform_xformEditor.xml`의 `jwf_XFormTemplate` 테이블을 활용하여 공통 양식 템플릿(문서 속성 8종 포함) 구조를 먼저 설계하고, 양식 확정 후 `createHTMLFormFile()`/`createJSFormFile()`로 개별 양식을 빠르게 적용하는 2단계 접근을 권장합니다.
- xFormEditor의 템플릿 복제(`insertTemplateData`) 기능을 활용하여 유사 양식 간 개발 효율을 높일 수 있습니다.
