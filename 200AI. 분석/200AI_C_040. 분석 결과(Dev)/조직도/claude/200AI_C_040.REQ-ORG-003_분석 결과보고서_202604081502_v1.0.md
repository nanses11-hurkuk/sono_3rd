#### 분석 범위

REQ-ORG-003의 분석 범위는 다음과 같다.

1. **SaaS 멀티 도메인 가시성 설정**: `sys_object_domain` 테이블 및 `DomainID=0`(그룹사) 기반 통합 조직도 트리 구성 기능을 활용하여 V1·V2 양사 조직도를 통합 조회할 수 있도록 한다.
2. **`isSaaS=Y` 파라미터 적용 및 JDBC 버그 수정**: SYNC FieldMapping.sql에서 GroupCode에 `CompanyCode_` 접두어를 붙여 V1·V2 간 코드 충돌을 방지하고, QUERY 타입 JDBC 바인딩 오류를 Java 서비스 단 사전 치환 로직으로 해결한다.
3. **전자결재 결재선 팝업 확장**: 결재선·참조 선택 시 `isDisplayAllDomain=true` 파라미터로 V1·V2 통합 조직도를 표시한다.
4. **이음톡·포탈 연동 API 확장**: 멀티 도메인 파라미터를 추가하여 양사 통합 데이터를 반환한다.
5. **ACL 정책 구현**: 계열사 간 정보 노출 범위를 고객과 협의하여 정책 반영한다.

#### 제약 사항

1. **`isSaaS` 파라미터 JDBC 버그 (즉시 수정 필요)**: 현재 FieldMapping.sql의 MemberOf, GroupCode QUERY 타입 sourceField 내 `#{_parameter[isSaaS]}` 표현식이 MariaDB Connector/J의 `INSERT INTO ... SELECT` 구문에서 다중 파라미터 바인딩 오류(`SQLSyntaxErrorException`)를 유발한다. 이 버그가 수정되지 않으면 `isSaaS=N` 환경에서 SYNC 배치가 실패한다. (산출 근거: `AI_Folder\item\SYNC\FieldMapping.sql` 분석, Claude 분석 수행보고서 (3-2))

2. **FieldMapping.sql CONSTANT 타입 빈 문자열 버그 (수정 완료)**: GroupPath, OUPath, SortPath 등 CONSTANT 타입에서 빈 문자열 값이 `'\''`로 잘못 저장되어 SQL 구조가 깨지는 현상. 수정 완료되었으나 DB 재삽입 및 재검증 필요. (산출 근거: `AI_Folder\item\SYNC\FieldMapping.sql` 분석)

3. **계열사 간 정보 노출 범위 미확정**: 타 계열사 사용자의 전화번호·메일·사진 노출 범위를 고객과 협의해야 한다. 미확정 시 '기본 정보(이름, 소속부서, 직위)만 노출'을 기본값으로 임시 구현한다. (산출 근거: Claude 분석 수행보고서 (3-2))

4. **통합 조직도 성능 저하 가능성**: V1+V2 통합 조직도 로드 시 단일 도메인 대비 데이터량이 2배 증가할 수 있다. 초기 로드 시 레벨 1~2만 반환하고 하위는 Lazy Loading으로 처리해야 한다. (산출 근거: Claude 분석 수행보고서 (3-2))

5. **이음톡 모듈 수정 범위 미파악**: 이음톡 연동 모듈의 코드 레벨 분석 필요. 수정 범위는 모듈 코드 확인 후 재산정 필요. (산출 근거: Claude 분석 수행보고서 (3-2))

6. **전자결재 결재선 정합성**: 타 계열사 사용자를 결재선에 지정할 때 결재 경로(결재 권한) 검증이 필수적이다. (산출 근거: Gemini 분석 수행보고서 (3))
