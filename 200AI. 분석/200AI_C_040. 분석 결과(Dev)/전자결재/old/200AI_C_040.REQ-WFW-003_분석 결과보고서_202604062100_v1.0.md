### 2-3. 기술적 타당성

#### 구현 난이도 / 제약 사항

구현 난이도는 **중간**입니다. `DicHelper`/`Fn_BaseGetDictionary_S()`/`RedisDataUtil` 기반 다국어 체계가 전자결재 모듈 전반에 이미 구현되어 있어, 베트남어 언어 팩 추가 및 사전 데이터 등록 방식으로 구현이 가능합니다. 다만, 다음 제약 사항이 존재합니다.

- 요구사항정의서에 "본문내용의 언어는 변환 불가함" 명시 → 고객사 사전 안내 필요
- 베트남어 번역 데이터 정확성 검증을 위한 고객사 검수 기간 필요
- `AutoApprovalLineService`의 `@Service`(Singleton) 스코프에서 `SessionHelper.getSession("lang")` 클래스 필드 선언 → 동적 반영 이슈 가능성 검증 필요
- "신청" 유형 관련 항목이 "검토 후 결정" 상태로 번역 대상 확정 대기

#### 대안 제시

1단계로 기존 다국어 체계에 베트남어 언어 코드 추가 및 시스템 공통 영역 다국어 적용, 2단계로 양식별 다국어 적용 및 고객사 검수를 권장합니다. 본문 번역이 향후 요구될 경우 외부 번역 API 연동을 별도 요구사항으로 관리합니다.

**산출 근거**: 분석 수행보고서(200AI_C_030.REQ-WFW-003_v1.0) 기술적 타당성
