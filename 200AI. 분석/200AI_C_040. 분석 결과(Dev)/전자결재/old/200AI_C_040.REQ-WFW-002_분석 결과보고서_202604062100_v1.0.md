### 2-3. 기술적 타당성

#### 구현 난이도 / 제약 사항

개별 양식당 구현 난이도는 **낮은** 수준입니다. `XFormEditorSvc`에 양식 생성/관리 표준 인터페이스가 이미 구현되어 있고, Jsoup 기반 HTML 파싱/편집 구조를 활용할 수 있습니다. 다만, 양식 종류/규격이 미확정(15종)이므로 고객사 양식 확정 일정이 전체 개발 일정에 직접적 영향을 미칩니다. 15종 양식의 FormPrefix 명명 규칙 사전 정의 및 문서번호 발번 규칙 확정이 필요합니다.

#### 대안 제시

`jwf_XFormTemplate` 공통 템플릿(문서 속성 8종 포함)을 먼저 설계하고, 양식 확정 후 개별 양식을 빠르게 적용하는 2단계 접근을 권장합니다. 템플릿 복제(`insertTemplateData`) 기능으로 유사 양식 간 개발 효율을 높일 수 있습니다.

**산출 근거**: 분석 수행보고서(200AI_C_030.REQ-WFW-002_v1.0) 기술적 타당성
