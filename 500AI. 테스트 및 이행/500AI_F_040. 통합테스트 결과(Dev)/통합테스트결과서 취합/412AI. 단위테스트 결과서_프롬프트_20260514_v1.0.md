# 프롬프트: 요구사항 ID별 산출물 이행 현황 체크 (계층형 헤더 매트릭스)

## 1. 목적
본 프롬프트는 프로젝트의 모든 요구사항(ID)에 대하여 각 업무의 단위테스트 결과서를 취합하고, 이를 **단계별 구분이 명확한 가로 확장형 매트릭스**로 정리하는 데 목적이 있다.

## 2. 대상 경로 및 파일
 `400AI. 구현\400AI_E_120. 단위테스트 결과(Dev)\4.단위테스트결과서취합본\단위테스트시나리오결과서 취합본.md`


## 3. 수행 지침
1. **ID 추출:** `C:\00.PROJECT\강민석_538 유티아이 베트남 그룹웨어 시스템 구축\200AI. 분석\230AI. 요구사항 정의\231.유티아이베트남 그룹웨어 시스템 구축_요구사항정의 및 GAP분석서_v1.5_20260414.md`에서 요구사항 ID를 기준으로 리스트를 저장해.
2. **2단 헤더 구성:** 상단에 단계를 표시하고, 하단에 상세 항목을 배치하여 시각적으로 구분하라.
3. **상태 표시:**
   - `O`: 문서가 존재하고 내용이 확인됨
   - `△`: 문서가 존재하나 내용 보완 필요
   - `X`: 문서가 존재하지 않음
   - `-`: 해당 없음
4. 요구사항별 상태 표시가 없는 항목은 요구사항 ID만 기입해줘
5. **결과 파일 생성:** 모든 체크가 완료되면 최종 결과물을 **Markdown(.md) 파일**로 생성하라.
   - **저장 경로:** `400AI. 구현\400AI_E_120. 단위테스트 결과(Dev)\4.단위테스트결과서취합본`
   - **파일 이름 형식:** `040AI. 단위테스트결과서_매트릭스_[년월일시분]_v[일련변호].md` (기존 파일이 존재할 경우 버전을 0.1씩 올린다.)
6. '대상 경로 및 파일' 으로 지정한 경로 이외에 Folder 는 read 하지 말것.
