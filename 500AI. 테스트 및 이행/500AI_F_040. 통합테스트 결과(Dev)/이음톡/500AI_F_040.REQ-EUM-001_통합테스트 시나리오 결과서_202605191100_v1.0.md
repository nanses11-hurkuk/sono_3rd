## (2) 개발 통합테스트 결과 내용

### 2-1. 목적 및 범위

- **목적**: 외부 vendor 사용자의 이음톡 접근 인증 흐름(VendorAuthPlugin → JWT 발행 → WebSocket 세션 → 채널 권한 필터) 시나리오별 통합 동작 검증 결과를 PM 보고 및 Staging 이관 기준 자료로 기록
- **범위**: `POST /api/eum/auth/external` → VendorAuthPlugin → EUM_VENDOR_USER_MAP → Redis ACL(TTL 2h) → WebSocket 채널 접근 전 구간. 소스 수정 불가로 외부 I/F·설정·DB 데이터 기반 검증.
- **제외**: 외부 Vendor API 실 환경 미구성으로 TC-001 실 연동 검증 제외 — WireMock 대체 수행, 별도 재수행 예정

### 2-5. 연동 흐름

```
[Vendor 클라이언트]
    POST /api/eum/auth/external
    Body: { vendorId, extUserId, secret }
        ↓
[이음톡 - VendorAuthPlugin]
    (Smarts4j CustomAuthenticator 구현체)
    ├─ 외부 Vendor API 검증 (WireMock 대체)
    ├─ EUM_VENDOR_USER_MAP 조회
    └─ 채팅 접근 권한 체크
        ↓
[JWT 발행 + WebSocket Endpoint URL 응답]
        ↓
[Redis] ACL 세션 저장 (TTL 2h)
        ↓
[WebSocket 서버] 채널 권한 필터 적용
```
