## (2) 개발 통합테스트 결과 내용

### 2-1. 목적 및 범위

- **목적**: V1(UTI VINA) ↔ V2(UTI VINA Vinh Phuc) 계열사 간 이음톡 채팅 메시지 라우팅(Redis Pub/Sub + EumMessageRouter) 시나리오별 통합 동작 검증 결과를 PM 보고 및 Staging 이관 기준 자료로 기록
- **범위**: V1 송신 → EumMessageRouter(isCrossTenant() 분기) → Redis Topic(`eum.routing.[TenantID].[RoomID]`) → V2 수신 전 구간. MongoDB 저장·Redis 검색 인덱스·다국어(베트남어) 정합성 포함. 소스 수정 불가로 설정·환경 기반 검증.
- **제외**: TC-005(V2 오프라인 메시지 재전달) — 재전달 정책 미정의로 보류
