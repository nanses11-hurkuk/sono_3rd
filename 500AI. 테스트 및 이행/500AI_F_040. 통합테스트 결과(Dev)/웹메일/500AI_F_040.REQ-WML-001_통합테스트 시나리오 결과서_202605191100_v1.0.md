## (2) 개발 통합테스트 결과 내용

### 2-1. 목적 및 범위

- **목적**: Exchange 기반 웹메일(covimail ↔ smarts4j_coviexchange) 전 계층 통합 동작 검증 결과를 시나리오별로 기록하여 PM 보고 및 Staging 이관 기준 자료로 활용
- **범위**: 메일 발송·수신·폴더 관리 → SvcImplFactory(domainId 분기, GAP-WML-001-10) → RestMailCon(EWS 50+ 엔드포인트) → Exchange Server 전 구간. AccessToken 갱신(GAP-WML-001-02), 대용량 첨부(GAP-WML-001-05), 구독 갱신 스케줄러(GAP-WML-001-09), 다국어(베트남어) UTF-8 정합성 포함
- **제외**: GAP-WML-001-02(AccessToken 자동 갱신) 실 Azure AD 연동 구간 — 별도 재수행 보고서 발행 예정

### 2-5. 연동 흐름

```
[브라우저 - 웹메일 UI]
        ↓ HTTP(S)
[covimail - MailConAjax]
    domainId 헤더 전달
        ↓
[SvcImplFactory]
    domainId 기반 런타임 분기
    ├─ MsMailSvcImpl  (Exchange 계열사 V1/V2)
    └─ MailSvcImpl    (IMAP — Exchange 비대상)
        ↓
[RestMailCon] → 50+ 엔드포인트
        ↓ EWS SOAP/HTTPS
[EWSCommon] → Bearer 토큰 인증 (GAP-WML-001-02 주석 해제 예정)
        ↓
[Exchange Server]
    메일 CRUD / 폴더 / 구독 / 카테고리 / 용량

[ResourceManager] ↔ [Redis]
    토큰 캐시 / 구독 TTL / 업로드 세션 키

[SubscriptionRenewalScheduler]
    만료 30분 전 자동 갱신 → Redis TTL 갱신
```
