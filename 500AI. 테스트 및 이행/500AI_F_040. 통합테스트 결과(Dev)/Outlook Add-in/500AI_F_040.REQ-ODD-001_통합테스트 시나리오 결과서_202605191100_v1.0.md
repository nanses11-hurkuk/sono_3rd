## (2) 개발 통합테스트 결과 내용

### 2-1. 목적 및 범위

- **목적**: Outlook Add-in(CAUTK → `/outlook/service.do` → OutlookAddinSvcImpl → CoviMSAPI → smarts4j_coviexchange → AD/Exchange) 전 계층 통합 동작 검증 결과를 시나리오별로 기록하여 PM 보고 및 Staging 이관 기준 자료로 활용
- **범위**: 비동기 조직도(G-06), 연락처 4종(G-04/G-07), 배포 그룹 CRUD(G-03), 세션 재연결(G-01), Redis Distributed Lock(G-05), 대용량 첨부(GETBIGATTACH), 다국어(베트남어) UTF-8 정합성
- **제외**: G-02(OAuth2AuthStrategy) 실 Azure AD 연동 구간 — 별도 재수행 보고서 발행 예정

### 2-5. 연동 흐름

```
[Outlook 클라이언트]
    CAUTK 파라미터: {mailAddress}|{timestamp}|{type}|{authKey}|[opType]|[json]
        ↓ HTTP(S) POST /outlook/service.do
[covimail - OutlookAddinCon]
    type별 라우팅 (GETORGCHART / GETCONTACTS / MANAGEDISTGROUP / GETBIGATTACH)
        ↓
[OutlookAddinSvcImpl]
    ├─ getOrgChartAsync()     → @Async(outlookAsyncExecutor) → Redis 결과 저장 → 폴링 API
    ├─ getContacts()          → CoviMSAPI.contact() → smarts4j_coviexchange RestContactCon
    ├─ manageDistGroup()      → CoviMSAPI.org() → smarts4j_coviexchange RestGroupCon
    └─ ExchangeSessionManager → LdapManager / EWSCommon (G-01 재연결 래핑)
        ↓ 내부 REST
[smarts4j_coviexchange]
    RestGroupCon  → LdapManager(LDAP) + ExchGroupSvcImpl(Exchange PowerShell)
    RestContactCon → ContactSvcImpl(EWS SOAP)
        ↓
[Active Directory / Exchange Server]
```
