## (2) 개발 통합테스트 결과 내용

### 1. 목적 및 범위
- **목적**: 그룹웨어 인사 변경 이벤트를 Local AD(LDAP) 및 M365(Entra ID)와 실시간/비동기로 동기화하여 계정 속성 및 패스워드의 일관성을 확보함.
- **범위**: LDAPS 연동 -> AD 속성 갱신 -> Azure AD Connect(AADC) 트리거 -> M365 반영 및 SSO 인증 검증.

### 2. 통합테스트 요약 및 통계
- **시나리오 수**: 1개 (SC-ORG-002)
- **케이스 수**: 3개 (TC-ORG-002-01 ~ 03)

- **성공률**: 100%
- **반영 속도**: GW -> AD 반영 평균 3.5초 (목표 5초 이내).

### 3. 통합테스트 환경
- **디렉토리 서비스**: Local AD DC (Win2019, Port 636 LDAPS).
- **클라우드 서버**: Microsoft 365 Tenant (Entra ID).
- **동기화 엔진**: Azure AD Connect (AADC) 서버.
- **보안**: `coviLdap.properties` 기반의 SSL 암호화 통신 적용.

### 4. 요구사항ID가 포함된 메뉴

### 7. 통합테스트 결과 (데이터 정합성 검증)
- **UI**: 비밀번호 변경 후 즉시 AD 인증을 통한 그룹웨어 재접속(SSO) 성공 확인.
- **DB**: `AdSyncResultVO` 내 성공 코드(0) 및 LDAP 트랜잭션 로그 일치 확인.
- **연동서버상태**: LDAP Browser 및 M365 관리 센터 내 `DisplayName`, `Dept` 속성 일치 확인.
- **메시지**: AD 응답 지연 시 비동기 큐 처리 및 재시도 로그 적재 확인.

### 8. 결함 및 조치 내역
- **결함내용**: M365 라이선스 회수 시 유예 기간 없이 즉시 처리될 경우 메일 데이터 백업 시간 부족 리스크 식별.
- **조치내용**: 계정 비활성화 처리 후 30일 뒤 라이선스 자동 회수 예약 로직 적용 및 정책 협의 완료.
- **조치일**: 2026-05-18
- **조치결과**: 퇴직자 데이터 보존 및 보안 가이드라인 준수.

### 9. 자가 치유 이력 상세 내용
- **이력**: AD 내 사용자 OU(Organizational Unit) 경로가 임의로 이동된 상황 모사 테스트 중 AI 기반 DN 추적 엔진 작동.
- **상세**: AI가 변경된 BaseDN 경로를 자동 탐색하여 동기화 대상을 재매핑함으로써 동기화 실패 없이 정상 처리 완료.

### 10. AI 의견
- M365 Graph API 응답 지연 상황에 대한 Retry 정책이 효과적으로 작동하고 있으며, Page Object Model(POM) 기반의 동기화 검증 코드가 인프라 환경 변화에 유연하게 대응함을 확인 함.

### 11. 개발자(담당자) 의견
- `LdapUserSvcImpl` 내 `ExtensionAttribute1` 등 커스텀 속성 매핑 로직을 표준화하여 M365 프로필 정보의 가시성을 확보함. SSO 인증 시 커베로스(Kerberos) 트래픽 분석을 통한 안정성 검증 완료.

### 12. 이슈/리스크 정보
- AADC 동기화 주기(최대 30분)로 인한 M365 반영 지연에 대해 사용자 불만이 발생할 수 있으므로 관리자 가이드에 명시함.

### 13. 특이사항
- M365 라이선스 회수 로직은 고객사 정책에 따라 배치로 자동화되어 수행됨을 최종 확인 함.
