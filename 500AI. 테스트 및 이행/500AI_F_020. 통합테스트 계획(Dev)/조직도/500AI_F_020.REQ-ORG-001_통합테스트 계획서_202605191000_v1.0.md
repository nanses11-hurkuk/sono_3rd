## (2) 통합테스트 계획 범위

### 1. 통합테스트 목적 및 범위
- **목적**: V2 계열사(영림원 ERP) 인사 정보 자동 동기화 기능과 V1 계열사 수동 관리 기능이 그룹웨어 DB 및 AD에 정확히 반영되는지 검증하고, 계열사간 데이터 정합성을 확보함.
- **난이도**: 중
- **범위**: ERP View 연동(MSSQL) -> GW 스테이징 -> GW 마스터 반영 -> AD/M365 연계 트리거 및 캐시 갱신 전 과정.

### 2. 개발자(담당자) 배정
- **담당 개발자**: 이윤정 (단위테스트 담당자)
- **QA 파트**: 솔루션 품질관리팀 배정 요청

### 3. 계획일 (시작/종료일)
- **계획일**: 2026-05-18 ~ 2026-05-22

### 4. 통합테스트 가능 여부
- **여부**: 가능 (단위테스트 완료 및 ERP View 접속 권한 확보됨)

### 5. 통합테스트 환경 및 인프라 구축 계획
- **인프라**: 그룹웨어 개발 서버 (Win2019, JDK 1.8), Redis 캐시 서버.
- **환경 구성**: Multi-DataSource(MSSQL JDBC) 연결 설정 및 ERP 접속용 방화벽 허용 확인.

### 6. 통합테스트 데이터 전략
- **전략**: 영림원 ERP 테스트 데이터(200건) 기반 실 데이터 매핑 테스트.
- **방법**: EmpNo를 식별키로 하여 UPSERT 로직(신규/수정/퇴직)의 정확도 100% 검증.

### 7. 통합테스트 상세 범위 계획
- **UI ↔ API Layer**: 관리자 페이지 동기화 트리거 API(`syncERPNow.do`) 호출 및 결과 JSON 수신.
- **Service ↔ Service**: `HRSyncSvc` -> `OrganizationManageSvc` 간의 사번/로그인ID 유효성 검사 및 데이터 전달.
- **App ↔ Database**: MyBatis Mapper를 통한 MSSQL(View) 조회 및 MariaDB(GW) `sys_object_user` 반영.
- **Async Tasks**: `CoviJobScheduler`를 이용한 새벽 1시 자동 동기화 배치 실행 및 로그 기록 검증.

| 시나리오 ID | 핵심 검증 내용 | 페르소나 | 통과 기준 | 멀티 유저 시나리오 | 영향도 |
| :--- | :--- | :--- | :--- | :--- | :--- |

### 9. AI 기반 통합테스트 실행 전략 계획
- **시나리오 합성**: ERP 데이터 변동 패턴(입사/이동/퇴직)을 조합하여 복합 시나리오 생성.
- **스크립트 자율 생성**: Playwright 기반의 UI 동기화 버튼 클릭 및 결과 로그 확인 스크립트 생성.
- **자가 치유 (Self-Healing)**: ERP View 스키마 변경 시 AI가 매핑 쿼리를 감지하여 테스트 코드 자동 보정.

### 10. 기본 코드 녹화 (Recording) 계획
- 관리자 동기화 설정 및 수동 실행 프로세스를 동영상 및 스크린샷으로 기록하여 결과 증적 확보.

### 11. AI를 이용한 스크립트 고도화 (Refactoring to POM) 계획
- 조직도 관리자 페이지 요소(Selector)를 Page Object Model로 추상화하여 유지보수성 강화.

### 12. AI 데이터 모킹 및 네트워크 인터셉션 (Data Mocking) 계획
- 영림원 ERP 서버 점검 대비 Mock API를 구축하여 비정상 데이터(사번 누락 등) 유입 시 예외 처리 검증.

### 13. 검증 및 디버깅 (Local Execution & Debug) 계획
- 로컬 개발 환경에서 동기화 Trace Log를 실시간 모니터링하며 데이터 변환 로직(`JobLevel` 변환 등) 상세 디버깅.

### 14. CI/CD 파이프라인 통합 및 모니터링 (CI Integration) 계획
- Jenkins 파이프라인에 통합 테스트 단계를 추가하여 배포 전 ERP 연동 정합성 자동 체크.

### 15. 성공 지표 및 대시보드 계획
- **성능**: 200건 동기화 30초 이내 완료.
- **정확도**: ERP 데이터와 GW DB 간의 데이터 일치율 100%.
- **가시성**: 관리자 페이지 내 동기화 이력(성공/실패 건수) 대시보드 노출.

### 16. 이슈/리스크 예상 정보
- 계열사간 중복 사번 발생 가능성 (EmpNo 유니크 체크 필수).
- 베트남 현지 네트워크 지연으로 인한 JDBC Connection Timeout 리스크.

### 17. 특이사항
- V1 계열사는 최초 1회 엑셀 이관 후 수동 관리가 원칙이므로, 자동 동기화 범위에서 제외되는지 명확히 필터링 확인 필요.
