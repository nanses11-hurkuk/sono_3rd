## (2) 통합테스트 계획 범위

### 1. 통합테스트 목적 및 범위
- **목적**: V1(UTI VINA) 및 V2(Vinh Phuc) 양 계열사의 임직원 정보가 포탈, 전자결재, 이음톡 등 주요 모듈 조직도에서 통합되어 정상 노출되는지 검증함.
- **난이도**: 중
- **범위**: `UnifiedOrgManager`를 통한 멀티 도메인 쿼리, `coviTree` 통합 렌더링, 모듈별(결재/채팅) 조직도 팝업 연동.

### 2. 개발자(담당자) 배정
- **담당 개발자**: 이윤정 (단위테스트 담당자)
- **QA 파트**: 솔루션 품질관리팀 배정 요청

### 3. 계획일 (시작/종료일)
- **계획일**: 2026-05-18 ~ 2026-05-22

### 4. 통합테스트 가능 여부
- **여부**: 가능 (V1/V2 데이터 마이그레이션 및 통합 쿼리 개발 완료)

### 5. 통합테스트 환경 및 인프라 구축 계획
- **인프라**: 그룹웨어 통합 개발 서버, MariaDB Cluster.
- **환경 구성**: `sys_domain` 테이블에 V1, V2 도메인 정보 등록 및 통합 조회 옵션(`IsGlobalOrgChart=Y`) 활성화.

### 6. 통합테스트 데이터 전략
- **전략**: 각 계열사별 50명 이상의 임직원 데이터를 구성하여 통합 트리 구조 테스트.
- **방법**: 동명이인(계열사만 다름) 케이스를 포함하여 UI상 구분 표기 확인.

### 7. 통합테스트 상세 범위 계획
- **UI ↔ API Layer**: `coviTree` UI 컴포넌트 -> `getUnifiedList.do` API 간의 JSON 데이터 수신 및 트리 로딩 성능 점검.
- **Service ↔ Service**: `UnifiedOrgManager` -> `OrgMapper` 간의 도메인 필터 우회 로직 검증.
- **App ↔ Database**: `CompanyCode` 기반의 그룹화 조회 및 Redis 통합 트리 캐싱(`ORG_UNIFIED_TREE_ALL`) 정합성 확인.
- **Async Tasks**: 캐시 갱신 시 전사 조직도 트리의 실시간 리로딩 여부 점검.

### 9. AI 기반 통합테스트 실행 전략 계획
- **시나리오 합성**: 계열사별 권한 필터가 적용된 상태에서 통합 조직도의 조회 범위가 의도대로 동작하는지 무작위 권한 테스트 수행.
- **스크립트 자율 생성**: 다양한 브라우저(Chrome, Edge, Mobile Web) 환경에서의 조직도 렌더링 호환성 스크립트 생성.
- **자가 치유 (Self-Healing)**: 도메인 추가/삭제 시 통합 쿼리 대상이 자동으로 갱신되는지 감지.

### 10. 기본 코드 녹화 (Recording) 계획
- 계열사 통합 검색 및 결재선 지정 과정을 동영상으로 기록하여 UX 시인성 검증.

### 11. AI를 이용한 스크립트 고도화 (Refactoring to POM) 계획
- `OrgChart.js` 팝업의 공통 요소들을 재사용 가능한 컴포넌트 모델로 리팩토링.

### 12. AI 데이터 모킹 및 네트워크 인터셉션 (Data Mocking) 계획
- 특정 계열사 DB 지연 상황을 모킹하여 조직도 로딩 시 타 계열사 데이터만이라도 우선 노출되는지(Partial Load) 검증.

### 13. 검증 및 디버깅 (Local Execution & Debug) 계획
- SQL 실행 계획 분석을 통한 통합 조회 쿼리의 성능 최적화 디버깅.

### 14. CI/CD 파이프라인 통합 및 모니터링 (CI Integration) 계획
- 배포 파이프라인에서 통합 조직도 API의 응답 스키마(계열사 코드 포함 여부) 자동 검증.

### 15. 성공 지표 및 대시보드 계획
- **로딩 속도**: 전사 트리 초기 로딩 1.5초 이내.
- **검색 정합성**: 사명 병기 표기율 100%.
- **사용자 만족도**: 결재선 지정 편의성 테스트 통과.

### 16. 이슈/리스크 예상 정보
- 대량의 전사 임직원 조회 시 브라우저 메모리 부하 (Lazy Loading 필수 적용).
- 계열사간 동일 부서명 존재 시 트리 시각적 혼동 리스크.

### 17. 특이사항
- `sys_base_config.IsGlobalOrgChart` 설정값에 따라 계열사 격리 모드와 통합 모드가 유연하게 전환되는지 집중 점검 필요.
