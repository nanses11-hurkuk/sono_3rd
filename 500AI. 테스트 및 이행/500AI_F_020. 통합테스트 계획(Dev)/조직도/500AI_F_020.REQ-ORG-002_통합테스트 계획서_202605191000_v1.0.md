## (2) 통합테스트 계획 범위

### 1. 통합테스트 목적 및 범위
- **목적**: 그룹웨어 인사 정보 변경 시 Local AD 및 Microsoft 365(Entra ID) 계정의 동기화 상태를 검증하고, 메일 사서함 및 라이선스 연동의 안정성을 확보함.
- **난이도**: 중
- **범위**: GW 인사 변경 이벤트 -> `ADSyncSvc` 호출 -> Local AD LDAP 반영 -> AADC를 통한 M365 동기화 확인.

### 2. 개발자(담당자) 배정
- **담당 개발자**: 이윤정 (단위테스트 담당자)
- **QA 파트**: 솔루션 품질관리팀 및 인프라 보안팀 배정 요청

### 3. 계획일 (시작/종료일)
- **계획일**: 2026-05-18 ~ 2026-05-22

### 4. 통합테스트 가능 여부
- **여부**: 가능 (coviexchange 모듈 구성 및 AD 접속 권한 확보됨)

### 5. 통합테스트 환경 및 인프라 구축 계획
- **인프라**: Local AD 서버 (Port 636 LDAPS), Azure AD Connect 서버, M365 테넌트.
- **환경 구성**: `coviLdap.properties` 내 AD 접속 정보 설정 및 LDAP SSL 인증서 설치 확인.

### 6. 통합테스트 데이터 전략
- **전략**: 테스트 계정(10건)을 이용한 속성(DisplayName, Dept, Password) 변경 테스트.
- **방법**: LDAP Browser 및 M365 관리 센터를 통한 실시간 속성 반영 결과 교차 검증.

### 7. 통합테스트 상세 범위 계획
- **UI ↔ API Layer**: GW 비밀번호 변경 UI -> `RestUserCon` API 호출 결과 확인.
- **Service ↔ Service**: `ADSyncSvc` -> `LdapUserSvc` -> `ExchangeRestProxy` 간의 연쇄 호출 성공 여부.
- **App ↔ Database**: AD 동기화 결과 로그(`AdSyncResultVO`)의 DB 기록 상태 점검.
- **Async Tasks**: AADC 동기화 주기(30분)에 따른 M365 반영 최종 지연 시간 모니터링.

### 9. AI 기반 통합테스트 실행 전략 계획
- **시나리오 합성**: 인사 정보 동기화(REQ-ORG-001)와 AD 동기화(REQ-ORG-002)를 연계한 End-to-End 시나리오 자동 생성.
- **스크립트 자율 생성**: LDAP 쿼리를 이용한 속성 검증 자동화 봇 스크립트 작성.
- **자가 치유 (Self-Healing)**: AD OU(Organizational Unit) 구조 변경 시 AI가 검색 BaseDN 경로 자동 탐색 및 보정.

### 10. 기본 코드 녹화 (Recording) 계획
- AD 및 M365 관리 콘솔에서의 데이터 반영 화면을 캡처하여 증적 문서화.

### 11. AI를 이용한 스크립트 고도화 (Refactoring to POM) 계획
- M365 Graph API 응답 처리 로직을 도메인 모델로 고도화하여 가독성 증대.

### 12. AI 데이터 모킹 및 네트워크 인터셉션 (Data Mocking) 계획
- AD 서버 다운 상황을 모킹하여 동기화 실패 시 GW DB 내 오류 로그 기록 및 재시도 큐 동작 검증.

### 13. 검증 및 디버깅 (Local Execution & Debug) 계획
- LDAP Bind 성공 여부 및 커베로스(Kerberos) 인증 트래픽 상세 로그 분석.

### 14. CI/CD 파이프라인 통합 및 모니터링 (CI Integration) 계획
- AD 동기화 모듈의 단위 기능 성공 여부를 빌드 파이프라인에서 체크.

### 15. 성공 지표 및 대시보드 계획
- **동기화 속도**: GW -> AD 반영 5초 이내.
- **가용성**: AD 동기화 성공률 99.9%.
- **모니터링**: `AdSyncResultVO` 기반의 동기화 현황 대시보드 구축.

### 16. 이슈/리스크 예상 정보
- AADC 동기화 지연(최대 30분)으로 인한 사용자 불만 발생 리스크 (안내 팝업 필요).
- 복잡한 Password 정책 불일치로 인한 AD 반영 실패 가능성.

### 17. 특이사항
- Local AD와 Entra ID 간의 싱크 구성은 고객사 인프라 영역이나, API 연동을 위한 M365 Graph API 권한 승인이 선행되어야 함.
