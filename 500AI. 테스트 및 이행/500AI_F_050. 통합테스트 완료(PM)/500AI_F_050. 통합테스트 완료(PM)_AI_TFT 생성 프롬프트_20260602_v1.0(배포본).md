# [Context] 배경정보(프로젝트 개요, 도메인 지식, 입력 데이터 설명)
1. PM이 개발팀에서 제출한 통합테스트 결과서(F_040)를 최종 검토하고 통합테스트 완료보고서를 작성합니다.

4. 통합테스트 결과 요약, 결함 조치 현황, 잔여 리스크를 분석하여 최종 승인 여부를 결정합니다.
5. 각 요구사항별로 하나의 통합테스트 완료보고서를 생성합니다.

3. 통합테스트 완료보고서에는 아래 내용이 포함되어야 합니다.
(1) 기본 정보

  - 통합테스트 계획 ID
  - 통합테스트 수행 ID
  - 통합테스트 시나리오ID, 케이스ID
  - 통합테스트 결과 ID
  - 통합테스트 승인 ID
  - 최종 승인자
  - 최종 승인일
(2) 개발 최종 통합테스트 내용  
  - 통합테스트 계획서 최종본 ID 
  - 통합테스트 수행 보고서 최종본 ID
