### 2-1. 단위테스트 목적 및 범위

#### 목적

REQ-EUM-003 신규 개발 11개 파일(`EumLocaleResolver`, `EumI18nService`, `EumI18nController`, `EumI18nMapper`, `EumUserLocaleMapper`, `i18n.js`, `font-vi.css`, `messages_vi.properties`, DDL 2개, DML 1개)의 기능 정확성을 API 블랙박스 검증 + UI 렌더링 육안 확인 + Redis CLI + 코드 리뷰 방식으로 검증한다. **소스코드 수정 불가 + 번역 검수 선행 조건** 이중 제약 하에서 TC-01~07, 12~14를 우선 수행하고, TC-08~11은 DML 배포(번역 검수 완료) 후 수행한다.

### 2-5. AI 의견

#### 전체 구조적 의견

1. **번역 검수 일정 조기 확정 권고**: TC-08~11은 번역 DML 배포 없이는 수행 불가하다. 번역 검수 일정을 PM 레벨에서 조기 확정하고, 검수 완료 즉시 DML 배포 → UI 검증을 연속 수행하는 계획을 수립해야 한다.

2. **TC-12(Oracle DDL) 즉시 수행**: 번역 검수 대기 중에도 TC-12는 즉시 수행 가능하다. `EUM_USER_PROFILE.LOCALE DEFAULT 'ko'` 기존 레코드 자동 적용 여부까지 반드시 확인한다.

3. **캐시 TTL 무제한 정책 위험 검토**: `eum:i18n:vi` Redis 캐시에 TTL 무제한 설정 시 번역 데이터 업데이트 시 캐시 무효화(`DEL eum:i18n:vi`) 수동 실행이 필수이다. 운영 절차서에 명시적으로 기재해야 한다.

4. **베트남어 원어민 참여 필수**: TC-09 성조 문자 깨짐 확인은 베트남어를 읽을 수 있는 원어민 또는 담당자가 반드시 참여해야 한다. 일반 개발자로는 성조 문자의 정확성을 판별하기 어렵다.

#### TC별 AI 기술 진단

| TC ID | AI 진단 의견 | 위험도 |
|-------|-----------|-------|
| TC-01 | DML 배포(번역 검수 완료) 전 API 호출 시 `MSG_VI` 컬럼값이 NULL일 수 있어 응답 JSON이 빈 맵으로 반환될 가능성. TC-01 수행 전 DML 배포 완료 여부 반드시 확인 | 중 |
