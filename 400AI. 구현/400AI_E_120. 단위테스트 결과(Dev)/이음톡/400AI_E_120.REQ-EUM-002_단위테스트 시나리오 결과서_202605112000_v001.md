### 2-5. AI 의견

#### 전체 구조적 의견

1. **PM 데이터 격리 정책 의사결정 최우선**: TC-01~08 전체가 블로킹되어 있다. PM 의사결정이 지연될수록 전체 일정이 후행된다. TC-09, 10을 즉시 수행하여 성과를 확보하고, PM 의사결정 가속화를 병행 요청해야 한다.

2. **V1 배포 후 V2 배포 순서 반드시 준수**: V2 배포 선행 시 V1의 라우팅 대상이 없어 `EumMessageRouter`에서 오류가 발생할 수 있다. 배포 체크리스트에 순서를 명시적으로 반영해야 한다.

3. **메시지 원문 로그 보안 리뷰 필수**: `EumMessagePayload.msg` 필드, `sender`, `target` 개인정보가 로그에 출력되지 않는지 TC-03 코드 리뷰에서 반드시 확인해야 한다.

4. **Redis Sorted Set 만료 정책 검토 권장**: `eum:msg:idx:{roomId}` Sorted Set이 지속 누적될 경우 Redis 메모리 압박이 발생할 수 있다. TC-05 수행 시 만료 정책(TTL 또는 주기적 정리 Job) 설계 여부를 추가로 확인하는 것을 권장한다.

#### TC별 AI 기술 진단

| TC ID | AI 진단 의견 | 위험도 |
|-------|-----------|-------|
| TC-01 | V1→V2 지연 ≤ 500ms는 개발 서버 환경에서 네트워크 상태에 따라 초과 가능. 기준값을 개발 서버 특성에 맞게 완화(예: 1,000ms) 검토 권장 | 중 |
| TC-03 | MongoDB 저장 실패 예외 처리 분기가 `try-catch` 외부에 있을 경우 Pub/Sub 발행이 차단되지 않을 수 있음. 코드 리뷰 시 예외 처리 위치 정확히 확인 | 상 |
| TC-08 | `allowed-routes` 설정이 `Set<String>`이라면 테넌트 조합 순서(`V1:V2` vs `V2:V1`)에 따라 차단 누락 가능성. 양방향 처리 여부 확인 | 중 |
| TC-09 | `ALTER TABLE` 실행 후 기존 채팅방 레코드 `ROOM_TYPE='INTERNAL'` 자동 적용 여부 반드시 확인 (`SELECT COUNT(*)` 비교) | 하 |
| TC-10 | `getIndexes()` 결과에서 `_id` 기본 인덱스 외 추가 2개 복합 인덱스가 정확히 `{roomId:1, timestamp:1}`, `{tenantId:1, timestamp:1}` 형식인지 확인 | 하 |
