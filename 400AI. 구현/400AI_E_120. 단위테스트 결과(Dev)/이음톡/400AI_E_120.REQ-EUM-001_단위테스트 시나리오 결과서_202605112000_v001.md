### 2-5. AI 의견

> **산출 근거**: 설계 상세 결과서(`300AI_D_040`), 단위테스트 계획서(`400AI_E_100`), 수행 보고서(`400AI_E_110`) 전 문서 재검증 기반 AI 기술 진단 의견

#### 전체 구조적 의견

1. **TC-EUM-001-12 보안 항목 최우선 수행**: `secret` 로그 출력 또는 JWT 키 하드코딩 발견 시 소스 수정 예외 적용이 필요한 유일한 상황이다. 다른 TC보다 먼저 수행하여 보안 결함 조기 탐지를 권고한다.

2. **외부 Vendor API 스텁 미확보 시 TC-01, 03, 04 블로킹**: 실 Vendor API 없이 개발 서버에서 스텁 URL을 설정할 수 있는지 PM + 인프라팀과 조기 협의가 필요하다. 설정 변경 자체는 소스 수정이 아니므로 허용 가능하다.

3. **WebSocket 권한 필터 5단계 처리 흐름 코드 리뷰 권장**: `EumChatChannelFilter`의 JWT 검증 → Redis ACL 조회 → 채널 권한 5단계 흐름은 TC-08~10 블랙박스 검증과 병행하여 코드 리뷰로도 확인하는 것을 권장한다.

4. **Redis 네임스페이스 충돌 사전 점검 필수**: 배포 전 `redis-cli SCAN 0 MATCH eum:acl:vendor:*` 실행으로 기존 키 잔류 여부를 반드시 확인해야 한다.

#### TC별 AI 기술 진단

| TC ID | AI 진단 의견 | 위험도 |
