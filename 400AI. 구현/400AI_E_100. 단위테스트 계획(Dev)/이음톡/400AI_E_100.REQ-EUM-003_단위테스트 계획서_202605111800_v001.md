#### 목적

REQ-EUM-003에서 신규 개발된 11개 파일의 기능 정확성을 검증한다. **소스코드 수정 불가** 제약으로 JUnit 화이트박스 테스트 대신 다음 방식으로 수행한다.

### 2-10. AI 기반 단위테스트 실행 전략 계획

- `lang=vi` API 응답·캐시·보안 시나리오 Claude 자동 합성 (Happy Path / Negative / Edge)

- `redis-cli` 스크립트: `EXISTS eum:i18n:vi`, `GET eum:i18n:vi` (개인정보 확인), `DEL eum:i18n:vi`
- UI 렌더링 체크리스트: 전 화면 항목별 Pass/Fail 기록 양식

#### 자가 치유 및 유지보수

- 번역 DML 추가 시 TC-08~11 재검증 자동 알림
- `messages_vi.properties` 키 추가 시 API 응답 JSON 키 추가 여부 자동 점검 체크리스트 갱신
