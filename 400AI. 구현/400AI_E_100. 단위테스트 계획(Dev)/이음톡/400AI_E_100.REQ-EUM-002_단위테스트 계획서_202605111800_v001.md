### 2-1. 단위테스트 목적 및 범위

#### 목적

REQ-EUM-002에서 신규 구현된 7개 파일(`EumMessageRouter`, `EumRedisIndexService`, `EumTenantRoutingFilter`, `EumMessage`, `EumMessageRepository`, `EumMessageBrokerConfig`, `EumMessagePayload`)의 기능 정확성을 검증한다. **소스코드 수정 불가** 제약으로 JUnit 화이트박스 테스트 대신 다음 방식으로 수행한다.

1. **종단간 메시지 흐름 검증(E2E Black-box)**: V1 → V2 실제 채팅 메시지 발송·수신 확인
2. **인프라 상태 직접 확인**: MongoDB 컬렉션·Redis Pub/Sub·Sorted Set 키 직접 조회
3. **기술적 코드 리뷰**: MongoDB 저장 실패 시 Pub/Sub 차단 로직, `@Async` 비동기 설정, 분산 락 TTL 확인
4. **설정 파일 검증**: `application.yml` `eum.routing.allowed-routes` 테넌트 조합 설정 확인

#### 범위

| 범위 구분 | 내용 |
|---------|------|
| **검증 대상** | 신규 7개 파일 + Oracle DDL(1개) + MongoDB 컬렉션 + Redis 키 구성 |
| **소스코드 수정** | **불가** |
| **검증 방식** | V1↔V2 E2E 채팅 메시지 발송 + MongoDB/Redis 직접 조회 + 코드 리뷰 |
| **계획 TC 수** | TC-EUM-002-01 ~ TC-EUM-002-10 (총 10건) |
| **인수 테스트** | 단위 검증 완료 후 별도 수행 (V1↔V2 실시간 전달 / 유실 없는 재전달 / 단일 테넌트 영향 없음) |

### 2-7. 단위테스트 데이터 전략
