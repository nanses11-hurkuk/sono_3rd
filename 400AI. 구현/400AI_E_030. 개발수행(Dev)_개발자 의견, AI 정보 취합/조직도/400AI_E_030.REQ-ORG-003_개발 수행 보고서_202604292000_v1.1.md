## (2) 개발 수행 검토 내용
- **AI 의견**: 
    - Smarts4j의 기본 멀티 테넌시 격리 원칙을 유지하면서 `CompanyCode=ALL` 파라미터를 통해 논리적 통합을 구현하는 'SaaS 확장 모델'이 최적임.
    - V1/V2 통합 시 데이터 양 증가로 인한 초기 렌더링 성능 저하를 방지하기 위해 `coviTree`의 Lazy Loading 옵션 적용이 필수적임.
- **개발자 의견**: 
    - 전자결재 결재선 지정 및 이음톡 대화 상대 검색 시 동명이인 혼선을 방지하기 위해 이름 앞 계열사 아이콘 또는 `[V1]` 접두어를 병기하는 UX 개선안을 적용하겠음.
    - `sys_base_config.IsGlobalOrgChart` 옵션을 통해 기능 활성화를 제어하여 추후 타 계열사 추가 시 유연하게 대응함.
- **분류**: 기능
- **변경 영향도 AI의견**: 
    - 조직도 조회 API(`getInitOrgTreeList.do`)의 도메인 필터 우회 시, 타 계열사 민감 정보(개인 휴대폰 번호 등)가 무단 노출될 위험이 있으므로 속성 레벨의 보안 필터링 영향도 검증 필요.
    - Redis 캐시 키 설계 시 `ORG_TREE:ALL` 전용 키를 도입하여 개별 도메인 캐시와의 충돌을 방지해야 함.
- **아키텍처**: 
    - **Domain Model**: `UnifiedOrgDTO` (id, pId, companyCode, type 포함)
    - **API Spec**: `GET /covicore/api/org/getUnifiedList.do` (IF-COV-USR-290 Extended)
    - **Logic Flow**: UI Call -> `domainID=ALL` 파라미터 -> `UnifiedOrgManager` -> `orgs_orgdept` Union Query -> Tree JSON 반환
    - **Security**: 도메인 간 가시성(Visibility) 허용 설정 (`sys_object_domain`) 연동
- **데이터베이스**: 
    - **Caching**: Redis Key `ORG_UNIFIED_TREE_ALL` 활용 및 1시간 단위 TTL 설정
    - **Migration**: 기존 `sys_object_user`, `sys_object_group` 내 `CompanyCode` PK 인덱스 최적화 확인
- **테스트 및 QA**: 
    - **Unit Test**: `UnifiedOrgManagerTest` - 멀티 도메인 쿼리 성능 및 결과 정합성 검증
    - **Test Coverage**: 통합 조직도 조회 및 필터링 로직 90% 이상
    - **인수 테스트 시나리오**: 
        1. V2 사용자가 전자결재 상신 시 V1 사용자를 결재선에 지정 가능한지 확인
        2. 이음톡에서 '전체' 검색 시 V1, V2 사용자가 통합 검색되는지 확인
        3. 통합 조직도 트리 로딩 속도가 2초 이내인지 확인 (Lazy Loading 검증)
- **인터페이스 (UI/UX)**: 
    - `coviTree` 컴포넌트 내 계열사별 로고 아이콘 표시 및 회사별 트리 노드 분리
    - 사용자 검색 결과 리스트에 소속 계열사 컬럼 추가
- **기술 스택 및 라이브러리 (Tech Stack)**: 
    - **Framework**: Spring Framework 5.x, MyBatis 3.x
    - **Persistence**: MariaDB (Unified View/Query)
    - **Database**: Redis (Global Org Cache)
    - **API Doc**: `IF_CORE_OrgApiCon.md` 기반 명세 확장
- **컴포넌트 설계**: 
    - **클래스 다이어그램**: `OrgApiCon` -> `UnifiedOrgManager` -> `OrgMapper`
    - **디자인 패턴**: Composite Pattern (부서-사용자 트리 구조), Decorator Pattern (회사명 병기 로직)
    - **Controller**: `OrgApiCon.java`, `MobileOrgCon.java` (기존 확장)
    - **Service**: `UnifiedOrgManagerImpl.java` (신규)
    - **Repository**: `UnifiedOrgMapper.xml`
    - **DTO**: `IntegratedOrgTreeVO`
- **비즈니스 로직**:

- **트랜잭션 및 예외 처리**: 
    - **Transactional**: `@Transactional(readOnly=true)` - 조회 중심이므로 DB 부하 최소화
    - **Custom Exception**: `DomainAccessDeniedException` (권한 없는 도메인 접근 시)
- **의존성 리스트**:

    - **모니터링/로깅**: Slow Query 발생 시 관리자 알림 및 Redis Hit Rate 모니터링
- **배포 및 롤백 가이드**:
