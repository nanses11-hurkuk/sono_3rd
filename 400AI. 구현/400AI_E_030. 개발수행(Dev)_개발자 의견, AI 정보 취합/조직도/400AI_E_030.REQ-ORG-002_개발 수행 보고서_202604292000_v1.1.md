## (2) 개발 수행 검토 내용
- **AI 의견**: 
    - GW를 인사 Master로 설정하고 AD를 Slave로 운영하는 '단방향 동기화' 전략이 데이터 무결성 확보에 가장 유리함.
    - AADC(Azure AD Connect) 전파 지연(최대 30분)을 사용자 가이드에 명시하여 '즉시 미반영'에 대한 클레임을 사전 방지해야 함.
- **개발자 의견**: 
    - `LdapManager.java`의 Connection Pooling을 최적화하여 베트남-한국 간 Latency 발생 시에도 LDAP Bind 실패를 방지하겠음.
    - M365 Graph API 호출 시 Throttling(할당량 제한)을 고려하여 Exponential Backoff 기반의 재시도 로직을 적용함.
- **분류**: 기능 / 인프라
- **변경 영향도 AI의견**: 
    - AD 계정 비활성화 시 연동된 Exchange 사서함 및 M365 라이선스 상태에 미치는 영향도가 매우 높으므로, 퇴사자 처리 시 OU 이동(`Retired` OU)을 우선 수행해야 함.
    - `sys_object_user` 내 `ExternalKey`(AD GUID) 정합성이 깨질 경우 SSO 로그인이 불가하므로 식별자 동기화 로직 검증 필수.
- **아키텍처**: 
    - **Domain Model**: `MSEWSUserDTO` (Exchange/AD 매핑 객체)
    - **API Spec**: `POST /v1/org/user/create`, `POST /v1/org/user/disable` (IF-EXCH-006)
    - **Logic Flow**: `OrgSyncCompleteEvent` -> `AdSyncEventHandler` -> `LdapTemplate` Bind/Modify -> AADC Sync
    - **Security**: LDAPS (Port 636) 및 Kerberos 인증 기반의 보안 채널 구성
- **데이터베이스**: 
    - **Caching**: AD GUID 및 UPN 정보를 Redis에 캐싱하여 로그인 시 AD 조회 횟수 절감
    - **Migration**: 기존 V1, V2 AD 사용자 GUID를 `sys_object_user.ExternalKey`로 전수 매핑(Update)
- **테스트 및 QA**: 
    - **Unit Test**: `LdapUserSvcTest` - LDAP 속성(sAMAccountName, UPN) 매핑 정확도 검증
    - **Test Coverage**: AD Object CRUD 핵심 메서드 80% 이상
    - **인수 테스트 시나리오**: 
        1. GW 사용자 생성 시 Local AD 계정 및 OU 자동 생성 확인
        2. GW 퇴사 처리 시 AD 계정 비활성화 및 `Retired` OU 이동 확인
        3. AD 암호 변경 시 GW SSO 로그인 연동 확인
- **인터페이스 (UI/UX)**: 
    - 관리자 > 조직관리 > 사용자 상세 정보 내 'AD 동기화 상태' 및 'M365 라이선스 정보' 탭 추가
    - AD 동기화 로그 뷰어 (성공/실패 상세 원인 표시)
- **기술 스택 및 라이브러리 (Tech Stack)**: 
    - **Framework**: Spring Framework 5.x, Spring LDAP
    - **Persistence**: LDAP (Directory Service)
    - **Database**: MariaDB (ExternalKey 저장)
    - **API Doc**: `INTERFACE_SPEC_EXCHANGE.md` 및 MS Graph API SDK Doc
- **컴포넌트 설계**: 
    - **클래스 다이어그램**: `AdSyncEventHandler` -> `LdapUserSvc` -> `LdapManager` -> `LdapTemplate`
    - **디자인 패턴**: Observer Pattern (인사 동기화 완료 이벤트 수신), Proxy Pattern (Graph API 호출 대행)
    - **Controller**: `RestUserCon.java` (coviexchange 재활용)
    - **Service**: `ADSyncSvcImpl.java` (신규), `LdapUserSvcImpl.java` (기존)
    - **DTO**: `AdUserRequestDTO`, `MSEWSUserDTO`
- **비즈니스 로직**: 
    - **시퀀스**: `SyncComplete` -> `pushToAD()` -> `renameOU()` -> `setUserAccountControl(514)` -> `syncToM365()`
- **트랜잭션 및 예외 처리**: 
    - **Transactional**: LDAP은 표준 DB 트랜잭션을 지원하지 않으므로, 작업 실패 시 Compensating Transaction(보상 트랜잭션) 수동 구현
    - **Custom Exception**: `LdapConnectionException`, `UPNConflictException`
- **의존성 리스트**: 
    - `spring-ldap-core`, `microsoft-graph`, `azure-identity`
- **테스트 및 성능**: 
    - **테스트 전략**: 개발용 AD 도메인 컨트롤러 및 Microsoft 365 Developer 테넌트 활용 통합 테스트
    - **모니터링/로깅**: `covi_smart4j.sys_log_ad_sync` 테이블 내 상세 LDAP 통신 로그 기록
- **배포 및 롤백 가이드**:
