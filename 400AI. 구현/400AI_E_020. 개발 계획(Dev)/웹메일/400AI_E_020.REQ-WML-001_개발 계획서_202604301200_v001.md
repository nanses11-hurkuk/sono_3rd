### 2-6. 개발 가능 여부

| GAP ID | 판단 | 근거 |
|--------|------|------|
| GAP-WML-001-01 (IMAP 타임아웃) | **즉시 가능** | 설계 내용이 확정되어 있으며 외부 선행 조건이 없다. |
| GAP-WML-001-02 (AccessToken 갱신) | **선행 조건 충족 후 가능** | Azure AD 앱 등록(`EWS.AccessAsApp`, `Mail.ReadWrite`) 확보 및 Exchange 서버 모던 인증 활성화 확인이 선행되어야 통합 테스트가 가능하다. |
| GAP-WML-001-03 (반복 일정) | **분석서 확보 후 가능** | 설계서에 분기 파라미터 방향은 확정되어 있으나 원본 분석서(`200AI_C_050`) 대조가 필요하다. |
| GAP-WML-001-04 (공유 메일함 필터) | **가능** | 설계 내용이 명확하며 소스코드 변경 범위가 제한적이다. |
| GAP-WML-001-05 (대용량 첨부) | **GAP-02 완료 후 가능** | 단위 테스트 수준 개발은 병렬 착수 가능하나, Graph API 통합 테스트는 `TokenRefreshService` 완료 후 수행한다. |
| GAP-WML-001-07 (Redis 동기화) | **GAP-02 완료 후 가능** | `ExchangeConfigDTO`의 OAuth2 필드가 포함된 상태로 진행되어야 하므로 GAP-02 완료 후 진행이 안전하다. |
| GAP-WML-001-08 (발신 제한 연동) | **GAP-02 완료 후 가능** | `setTransportRule.ps1`의 인증 방식이 GAP-02의 OAuth2 전환 결과와 연계된다. |
| GAP-WML-001-09 (웹훅 자동 갱신) | **가능 (GAP-02 병렬 착수 가능)** | 스케줄러 개발 자체는 병렬 착수 가능하며, EWS 구독 통합 테스트는 GAP-02 완료 후 수행한다. |
| GAP-WML-001-10 (SvcImplFactory) | **GAP-02 완료 후 가능** | `ResourceManager` 및 `ExchangeConfigDTO` 변경 완료 상태에서 진행이 안전하다. |
