## (2) 개발 프로그램 명세서 상세 내용

### 1. 데이터베이스 및 엔티티 설계 (REQ-ORG-003)
- **테이블 명세**:
    - `sys_object_domain`: 계열사별 도메인 정보 및 통합 조회 허용 여부 설정.
    - `orgs_orgdept`: `CompanyCode` 기반 멀티 테넌트 데이터 관리.
- **컬럼 상세**:
    - `DomainID` (int): 0(그룹사 통합) 및 개별 법인 ID.
    - `CompanyCode` (varchar(50)): 'V1', 'V2' 식별자.
- **캐시 설계**:
    - 통합 조직도 트리 캐시: `ORG_UNIFIED_TREE_ALL` (JSON String).
    - 캐시 만료 정책: 조직도 변경 이벤트 발생 시 전체 삭제 후 Lazy Reload.

### 2. 시스템 아키텍처 및 흐름 (REQ-ORG-003)
- **컴포넌트 다이어그램**:
    - `Common.js (coviTree)` -> `OrgApiCon` -> `UnifiedOrgManager` -> `Redis` (1차) / `MariaDB` (2차).
- **비즈니스 프로세스(BPM)**:
    1. 사용자가 조직도 팝업 호출.
    2. `domainID=ALL` 파라미터 전달.
    3. `UnifiedOrgManager`에서 `sys_domain` 설정 체크.
    4. V1, V2 최상위 노드부터 통합 트리 생성.
- **상태 코드 정의**:
    - `AUTH_OK`: 조회 권한 있음.
    - `ACL_DENIED`: 계열사 간 가시성 미설정 시 접근 거부.

### 3. API 및 인터페이스 명세 (REQ-ORG-003)
- **Endpoint**: `GET /covicore/api/org/getUnifiedList.do`
- **Request Header/Body**:
    - Params: `domainID=ALL`, `loadLevel=2`, `companyCode=V1|V2`
- **필드별 Validation 체크**:
    - `domainID`: 'ALL' 또는 유효한 Integer ID.
- **Response Body**:
    - `{ "list": [ { "id": "DEPT01", "pId": "ROOT", "name": "[V1] 인사팀", "companyCode": "V1" }, ... ] }`
