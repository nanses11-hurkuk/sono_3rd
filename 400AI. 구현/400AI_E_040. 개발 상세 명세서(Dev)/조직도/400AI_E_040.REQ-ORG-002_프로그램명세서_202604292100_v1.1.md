## (2) 개발 프로그램 명세서 상세 내용

### 1. 데이터베이스 및 엔티티 설계 (REQ-ORG-002)
- **테이블 명세**:
    - `sys_object_user`: AD 연동용 `ExternalKey`(GUID) 저장.
    - `sys_log_ad_sync`: AD 동기화 이력 및 상세 오류 로깅 테이블.
- **컬럼 상세**:
    - `ExternalKey` (varchar(100)): Local AD의 `objectGUID` 저장 필드.
    - `MailAddress` (varchar(100)): AD/M365 메일 주소 정합성 유지.
- **캐시 설계**:
    - 로그인 시 AD 연동 속성 재사용을 위해 Redis에 `AD_USER_MAP` 구조로 캐싱.

### 2. 시스템 아키텍처 및 흐름 (REQ-ORG-002)
- **컴포넌트 다이어그램**:
    - `AdSyncEventHandler` -> `LdapUserSvc` -> `LdapManager` -> `Local AD`
    - `Microsoft Graph SDK` -> `Entra ID` (M365 상태 정보 조회)
- **비즈니스 프로세스(BPM)**:
    1. GW 인사 변경 감지 (Spring Event).
    2. Local AD 계정 유무 확인.
    3. `LdapTemplate` 기반 Attribute 갱신 (displayName, mail, department).
    4. 퇴사 시 `Retired` OU로 이동 및 `userAccountControl=514` 설정.
- **상태 코드 정의**:
    - `AD_SYNC_OK` (S): 동기화 성공.
    - `AD_CONN_FAIL` (F1): LDAP 서버 연결 실패.
    - `UPN_CONFLICT` (F2): 이미 사용 중인 메일 주소.

### 3. API 및 인터페이스 명세 (REQ-ORG-002)
- **Endpoint**: `POST /v1/org/user/create` (Internal API)
- **Request Header/Body**:
    - Header: `Authorization: Internal-Key`, `domainId: UTI_AD`
    - Body: `{ "logonId": "leeyj", "displayName": "이윤정", "upn": "leeyj@utivina.com" }`
- **필드별 Validation 체크**:
    - `upn`: 필수값, RFC 5322 메일 규격 준수 여부.
- **Response Body**:
    - `{ "result": "SUCCESS", "guid": "8763-9821-..." }`

### 4. 핵심 로직 구현 상세 (REQ-ORG-002)
- **Service Layer 메서드**:
    - `pushToAD(UserDTO)`: 개별 사용자 AD 생성/수정 요청.
    - `disableADUser(String logonId)`: 퇴사자 계정 비활성화 및 OU 이동.
- **트랜잭션 관리**:
    - LDAP은 롤백이 불가하므로, 실패 시 GW 테이블의 `AD_IsUse` 플래그를 'N'으로 원복하는 보상 트랜잭션 수행.
- **비동기 처리**: AD 동기화는 대기 시간이 길어질 수 있으므로 `@Async`를 통한 백그라운드 처리.

### 5. 권한 및 보안 (REQ-ORG-002)
- **인가(Authorization)**: 서비스 계정(Admin Account) 기반 LDAP Bind 수행.
- **데이터 접근 권한**: AD 제어용 Internal API는 특정 IP(GW WAS)에서만 호출 허용.
- **개인정보 처리**: AD 속성 중 '전화번호' 등 개인 식별 정보 전송 시 SSL(Port 636) 암호화 필수.

### 6. 테스트 케이스 명세 (REQ-ORG-002)
- **단위 테스트(Unit Test)**:
    - `testLdapRename()`: 퇴사자 OU 이동 시 Path 정상 생성 여부.
    - `testPasswordPolicy()`: AD 암호 복잡도 정책 위반 시 에러 처리.
- **모킹(Mocking) 전략**: `LdapTemplate`을 모킹하여 네트워크 단절 시나리오 테스트.
