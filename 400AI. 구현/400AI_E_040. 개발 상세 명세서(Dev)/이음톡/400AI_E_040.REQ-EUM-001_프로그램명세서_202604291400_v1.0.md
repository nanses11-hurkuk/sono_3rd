**EumVendorUserMapper.java** — MyBatis Mapper 인터페이스:
```java
@Mapper
public interface EumVendorUserMapper {
    // VENDOR_ID + EXT_USER_ID로 사용자 조회 (로그인 시 사용)
    VendorUserMap findByVendorAndExtUser(
        @Param("vendorId")   String vendorId,
        @Param("extUserId")  String extUserId
    );
    // 신규 Vendor 사용자 등록
    int insert(VendorUserMap vendorUserMap);
    // 계정 상태(USE_YN) 변경
    int updateUseYn(@Param("id") String id, @Param("useYn") String useYn);
    // 만료일 갱신
    int updateExpireDt(@Param("id") String id, @Param("expireDt") Timestamp expireDt);
}
```

**EumVendorUserMapper.xml** — SQL 정의:
```xml
<mapper namespace="com.eum.auth.mapper.EumVendorUserMapper">

  <resultMap id="VendorUserMapResult" type="com.eum.auth.domain.VendorUserMap">
    <id     property="id"          column="ID"/>
    <result property="vendorId"    column="VENDOR_ID"/>
    <result property="extUserId"   column="EXT_USER_ID"/>
    <result property="pwdHash"     column="PWD_HASH"/>
    <result property="contact"     column="CONTACT"/>
    <result property="accessScope" column="ACCESS_SCOPE"/>
    <result property="useYn"       column="USE_YN"/>
    <result property="regDt"       column="REG_DT"/>
    <result property="expireDt"    column="EXPIRE_DT"/>
  </resultMap>

  <select id="findByVendorAndExtUser" resultMap="VendorUserMapResult">
    SELECT ID, VENDOR_ID, EXT_USER_ID, PWD_HASH, CONTACT,
           ACCESS_SCOPE, USE_YN, REG_DT, EXPIRE_DT
      FROM EUM_VENDOR_USER_MAP
     WHERE VENDOR_ID  = #{vendorId}
       AND EXT_USER_ID = #{extUserId}
  </select>

  <insert id="insert">
    INSERT INTO EUM_VENDOR_USER_MAP
      (ID, VENDOR_ID, EXT_USER_ID, PWD_HASH, CONTACT, ACCESS_SCOPE, USE_YN, REG_DT, EXPIRE_DT)
    VALUES
      (#{id}, #{vendorId}, #{extUserId}, #{pwdHash}, #{contact},
       #{accessScope}, #{useYn}, SYSTIMESTAMP, #{expireDt})
  </insert>

  <update id="updateUseYn">
    UPDATE EUM_VENDOR_USER_MAP
       SET USE_YN = #{useYn}
     WHERE ID = #{id}
  </update>

  <update id="updateExpireDt">
    UPDATE EUM_VENDOR_USER_MAP
       SET EXPIRE_DT = #{expireDt}
     WHERE ID = #{id}
  </update>

</mapper>
```


|--------|------|------|
| `accessToken` | String | JWT Access Token. HS256 서명. |
| `tokenType` | String | 토큰 유형. 항상 `"Bearer"` 고정. |
| `expiresIn` | Integer | 토큰 만료까지 남은 시간(초). Redis ACL TTL과 동일(7200). |
| `websocketEndpointUrl` | String | WebSocket 연결 대상 URL. WSS 프로토콜 적용. |

#### Endpoint 2 — WebSocket 핸드셰이크 (EumChatChannelFilter)

| 항목 | 내용 |
|------|------|
| **Protocol** | WebSocket (wss://) |
| **URL** | `wss://eum.example.com/ws/chat` |
| **인증 방식** | HTTP Upgrade 요청 헤더에 `Authorization: Bearer {accessToken}` 포함 |

**핸드셰이크 처리 흐름**:

1. `EumChatChannelFilter.beforeHandshake()` 진입.
2. `Authorization` 헤더에서 JWT 추출 및 서명 검증.
3. JWT 만료 여부 확인. 만료 시 Upgrade 요청 거부(HTTP 401).
4. JWT 내 `vendorId`, `extUserId` Claim 추출.
5. Redis에서 `eum:acl:vendor:{vendorId}:{extUserId}` 키 조회.
6. 요청 채널 ID가 `allowedChannels` 목록에 포함되어 있는지 확인. 미포함 시 거부(HTTP 403).
7. 검증 통과 시 `attributes` 맵에 사용자 정보 주입하여 핸드셰이크 허용.

#### Service Layer 메서드 상세

**VendorAuthPlugin.validate()**:
```java
public VendorAuthResult validate(VendorCredentialDto cred) {
    // 1. 외부 Vendor API 호출 (Timeout: 3초)
    boolean vendorVerified = callVendorApi(cred.getVendorId(),
                                           cred.getExtUserId(),
                                           cred.getSecret());
    if (!vendorVerified) throw new VendorAuthFailException();

    // 2. 내부 DB 계정 조회
    VendorUserMap user = vendorUserMapper
        .findByVendorAndExtUser(cred.getVendorId(), cred.getExtUserId());
    if (user == null)                          throw new VendorAuthFailException();
    if ("N".equals(user.getUseYn()))           throw new VendorUserDisabledException();
    if (isExpired(user.getExpireDt()))         throw new VendorUserExpiredException();

    // 3. 채팅 채널 접근 권한 확정
    SessionAcl acl = buildSessionAcl(user);

    // 4. Smarts4j 세션 Context에 외부 사용자 정보 주입
    injectSessionContext(cred.getVendorId(), cred.getExtUserId(), acl);

    // 5. JWT 생성
    String jwt = jwtTokenProvider.generateToken(cred.getVendorId(),
                                                cred.getExtUserId(), 7200);

    // 6. Redis 세션 ACL 저장 (TTL 7200초)
    sessionAclManager.save(cred.getVendorId(), cred.getExtUserId(), acl, 7200);

    return new VendorAuthResult(jwt, "Bearer", 7200, wsEndpointUrl);
}
```

**callVendorApi() — 외부 API 호출 (Timeout + Retry)**:
```java
private boolean callVendorApi(String vendorId, String extUserId, String secret) {
    // HttpClient Connection Timeout: 3,000ms, Read Timeout: 3,000ms
    // 실패 시 VendorApiTimeoutException 발생 (재시도 없음 — 인증은 멱등성 보장 어려움)
    try {
        ResponseEntity<VendorApiResponse> resp = vendorApiClient.verify(vendorId, extUserId, secret);
        return resp.getStatusCode().is2xxSuccessful() && resp.getBody().isVerified();
    } catch (ResourceAccessException e) {
        throw new VendorApiTimeoutException("Vendor API 응답 초과: " + vendorId);
    }
}
```

**EumSessionAclManager.save() / delete()**:
```java
public void save(String vendorId, String extUserId, SessionAcl acl, long ttlSeconds) {
    String key = buildKey(vendorId, extUserId);
    String json = objectMapper.writeValueAsString(acl);
    redisTemplate.opsForValue().set(key, json, ttlSeconds, TimeUnit.SECONDS);
}

public void delete(String vendorId, String extUserId) {
    redisTemplate.delete(buildKey(vendorId, extUserId));
}

private String buildKey(String vendorId, String extUserId) {
    return String.format("eum:acl:vendor:%s:%s", vendorId, extUserId);
}
```

#### 트랜잭션 관리

- `EumVendorUserMapper`를 사용하는 DB 조회는 읽기 전용 트랜잭션(`@Transactional(readOnly = true)`)으로 처리하여 불필요한 Lock을 방지한다.
- `updateUseYn()` 등 DB 쓰기 작업은 `@Transactional`(기본 설정, REQUIRED 전파)로 처리한다.
- Redis 저장은 트랜잭션 외부에서 처리한다. DB 커밋 완료 이후 Redis 저장을 수행하며, Redis 저장 실패 시 `@Retryable(maxAttempts = 3, backoff = @Backoff(delay = 500))` 재시도를 적용한다.

#### 비동기 처리

- `VendorAuthPlugin.validate()` 흐름 자체는 동기로 처리한다. 인증 결과(JWT)를 즉시 반환해야 하므로 비동기 적용 대상이 아니다.
- 계정 비활성화 이벤트(`USE_YN = 'N'` 업데이트) 후 Redis ACL 삭제는 이벤트 리스너(`@EventListener`) 또는 별도 `@Async` 메서드로 비동기 처리하여 DB 트랜잭션 완료 후 Redis 삭제가 후속 실행되도록 한다.

#### 데이터 접근 권한

- Vendor 사용자는 자신의 `ACCESS_SCOPE`에 등록된 채팅 채널에만 접근할 수 있다.
- 채팅 채널 접근 권한은 WebSocket 연결 수립 시 `EumChatChannelFilter`에서 Redis ACL을 기준으로 검증한다.
- 채팅 메시지 열람 범위는 자신이 접근 권한을 가진 채팅방 내 메시지로 한정된다. 채팅방 외 메시지 조회 시도는 서버 사이드에서 차단한다.
- 내부 사용자 조직도 및 개인정보는 Vendor 사용자에게 노출하지 않는다. 채팅 상대방 이름 표시 범위는 같은 채팅방 참여자로 제한한다.
