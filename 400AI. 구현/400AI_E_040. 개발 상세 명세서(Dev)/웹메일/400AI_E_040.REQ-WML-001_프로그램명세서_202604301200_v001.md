**[캐시 키 1] msAccessToken:{DomainId}** — GAP-WML-001-02

| 항목 | 내용 |
|------|------|
| **키 패턴** | `msAccessToken:{DomainId}` |
| **예시** | `msAccessToken:domain_vn01` |
| **저장 값 타입** | String (Azure AD AccessToken 원문 Bearer 토큰) |

| **캐시 히트 동작** | Redis에서 토큰 문자열을 그대로 반환한다. MSAL4J 호출이 발생하지 않는다. |
| **캐시 미스 동작** | MSAL4J `ConfidentialClientApplication.acquireToken(ClientCredentialParameters)`를 호출하여 신규 토큰을 발급한 후, Redis에 TTL 3,500초로 저장하고 반환한다. 최대 10초 대기한다. |
| **보안 주의사항** | 토큰 원문이 Redis에 평문으로 저장된다. Redis 접근 통제(방화벽, AUTH 인증, TLS)를 인프라팀이 보장해야 한다. 로그에 토큰 값을 절대 출력하지 않는다. |

#### 2-5-1. 인가 (Authorization)

| 항목 | 내용 |
|------|------|
| **EWS WebCredentials 방식** | Exchange 서버 사용자 계정(`ewsUser`, `ewsPwd`, `ewsDomain`)과 가장(Impersonation) 기반 인가. `ExchangeConfigDTO`에서 조회한 관리자 계정으로 대상 사용자를 가장하여 EWS를 호출한다. |
| **EWS OAuth2 방식 (신규)** | Azure AD 앱 등록(`EWS.AccessAsApp` 권한)을 통한 ClientCredentials Flow. 앱 수준의 인증으로 Exchange 사용자 가장(Impersonation)을 유지한다. `oauthClientId`, `oauthClientSecret`, `oauthTenantId`가 `ExchangeConfigDTO`에 저장된다. |
| **Graph API 방식** | Azure AD 앱 `Mail.ReadWrite` 권한으로 해당 사용자의 메일박스에 접근. AccessToken은 `TokenRefreshService`를 통해 발급·갱신된다. |
| **API 요청 인가** | 모든 `/v1/mail/**` 엔드포인트는 `domainId` 헤더를 필수로 요구한다. `domainId` 없이 요청 시 Exchange 설정을 조회할 수 없으므로 처리 불가. |

#### 2-5-2. 데이터 접근 권한

| 항목 | 내용 |
|------|------|
| **도메인 격리** | `ResourceManager`는 `domainId`를 키로 Exchange 설정을 분리 관리한다. A 도메인의 요청이 B 도메인 Exchange에 접근하는 것을 구조적으로 방지한다. `SvcImplFactory`도 `domainId` 기반으로 서비스 구현체를 선택하므로 도메인 격리가 유지된다. |
| **공유 메일함 접근** | `isSharedMailbox=true` 파라미터가 전달된 경우에만 공유 메일함 모드로 처리한다. Exchange 서버의 공유 메일함 접근 권한은 EWS Impersonation 또는 Full Access 권한 부여로 제어하며, 그룹웨어 애플리케이션은 이를 그대로 위임한다. |
| **Redis 접근 권한** | Redis에 저장된 `exchangeConfig` 내 `oauthClientSecret`, `oauthClientId` 등 민감 정보는 Redis 자체 AUTH 인증과 방화벽으로 보호한다. 애플리케이션 내부에서는 `RedisDataUtil`을 통해서만 접근한다. |

#### 2-5-3. 개인정보 처리

| 항목 | 내용 |
|------|------|
| **메일 데이터** | 메일 본문, 수신자, 첨부파일 등 개인정보 성격의 데이터는 그룹웨어 RDB에 저장되지 않고 Exchange/M365 서버에만 존재한다. 그룹웨어 서버는 요청 시에만 Exchange에서 조회하여 클라이언트에 전달하는 중계 역할을 수행한다. |
| **로그 마스킹** | `TokenRefreshService` 로그에 AccessToken 원문이 출력되지 않도록 한다. `ExchangeConfigDTO.toString()`에서 `oauthClientSecret`, `ewsPwd`, `adPwd` 등 민감 필드를 `"***"`로 마스킹하여 오버라이드한다. |
| **첨부파일 임시 데이터** | 청크 업로드 중 서버 메모리에 `byte[] chunk`가 일시적으로 존재하나, 업로드 완료 후 GC에 의해 해제된다. 파일 데이터를 서버 디스크에 임시 저장하지 않는다. |
| **Redis 내 메일 주소** | `ewsSubscription:{DomainId}:{userId}` 키에 `mailAddress`가 포함된다. 이는 구독 관리를 위한 기술적 필요에 의한 것이며, Redis 보안 설정으로 접근을 제한한다. |
