### ■ 배포 실행 요약 (Executive Summary)

**요구사항 개요**: 이음톡(EumTalk) / [이음톡] 외부 사용자 인증 플러그인 개발

**요구사항 상세 설명**: 외부 Vendor 사용자(Vendor사 직원)가 이음톡에 접속할 수 있도록 전용 인증 체계를 구현한다. Smarts4j `CustomAuthenticator` 인터페이스를 상속한 `VendorAuthPlugin`을 개발하고, 외부 계정 매핑 테이블(`EUM_VENDOR_USER_MAP`) 신설, Redis 세션 ACL(TTL 2시간) 관리, 채팅 채널 접근 권한 필터(`EumChatChannelFilter`) 구현을 포함한다.

| 항목 | 내용 |
|------|------|
| **수정 파일** | `EUM_VENDOR_USER_MAP_DDL.sql` (신규), `VendorUserMap.java` (신규), `EumVendorUserMapper.java` (신규), `EumVendorUserMapper.xml` (신규), `EumAuthController.java` (신규), `VendorCredentialDto.java` (신규), `VendorAuthPlugin.java` (신규), `EumSessionAclManager.java` (신규), `EumChatChannelFilter.java` (신규) — 총 9개 파일 신규 생성 |
| **DB 반영** | `EUM_VENDOR_USER_MAP` 테이블 신규 생성 (컬럼 9개, PK·UQ·CHK 제약, 복합 인덱스). 기존 테이블 변경 없음. |
| **테스트 통과율** | 자료없음 (단위 테스트 수행 후 결과 갱신 필요. 목표: 핵심 3개 클래스 90% 이상 커버리지) |
| **요구사항 충족** | 수용. 요구사항 기준(`REQ-EUM-001` — "vendor 사에서도 사용할 수 있도록 요청")으로 개발 계획 / 상세 명세 / 소스 변경 요청서 추적키 정합성 확인 완료. |

### ■ 기술적 변경 상세 (Technical Review)

**소스 코드 정합성 (Code Diff)**

| 구분 | Before | After |
|------|--------|-------|
| `VendorAuthPlugin.java` | 존재하지 않음 | 신규 생성. Smarts4j `CustomAuthenticator` 구현. 외부 Vendor API 호출(Timeout 3초) → DB 계정 조회 → JWT 발행 → Redis ACL 저장 전체 흐름 구현. `secret` 로그 출력 금지 처리 포함. |
| `EumAuthController.java` | 존재하지 않음 | 신규 생성. `POST /api/eum/auth/external` 엔드포인트. `@Valid` 입력값 검증, 글로벌 예외 핸들러 위임 구조. |
| `EumVendorUserMapper.xml` | 존재하지 않음 | 신규 생성. `findByVendorAndExtUser`, `insert`, `updateUseYn`, `updateExpireDt` 4개 SQL 정의. 복합 인덱스 활용 조회 쿼리 포함. |
| `EumSessionAclManager.java` | 존재하지 않음 | 신규 생성. Redis 키 패턴 `eum:acl:vendor:{vendorId}:{extUserId}`, TTL 7200초, `@Retryable(maxAttempts=3)` 적용. |
| `EumChatChannelFilter.java` | 존재하지 않음 | 신규 생성. `HandshakeInterceptor` 구현. JWT 검증 → Redis ACL 조회 → 채널 접근 권한 5단계 처리 흐름. |
| 기존 인증 모듈 | 내부 임직원 전용 인증 (`UserDetailsService`) | 변경 없음. Vendor 인증은 Extension Point로 독립 추가. 기존 인증 흐름 영향 없음 확인 필요. |

**DB 인프라 검증**

| 항목 | 내용 |
|------|------|
| DDL/엔티티 변경 | `EUM_VENDOR_USER_MAP` 신규 테이블 생성 확인. 기존 사용자 테이블(`EUM_USER_PROFILE` 등) 변경 없음. |
| 마이그레이션 검증 | 스테이징 환경 DDL 실행 → 테이블 구조, 제약조건(PK, UQ, CHK), 인덱스(`IDX_EUM_VENDOR_USER`) 생성 정상 여부 확인 필요 |
| Redis 인프라 | `eum:acl:vendor:*` 네임스페이스 키 기존 캐시 키 충돌 여부 확인 필요. TTL 7200초 정상 설정 여부 확인 필요. |
| 롤백 스크립트 | `DROP TABLE EUM_VENDOR_USER_MAP;` 스크립트 별도 보관. Redis 키는 TTL 만료 또는 `redis-cli SCAN 0 MATCH eum:acl:vendor:*` 패턴 삭제. |
