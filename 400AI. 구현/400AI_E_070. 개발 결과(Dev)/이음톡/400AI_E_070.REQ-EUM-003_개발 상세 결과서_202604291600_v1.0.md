### ■ 배포 실행 요약 (Executive Summary)

**요구사항 개요**: 이음톡(EumTalk) / [이음톡] 베트남어 다국어 대응 및 폰트 고도화

**요구사항 상세 설명**: 이음톡 서비스 전반에 걸쳐 베트남어(`vi`) 로케일을 적용한다. `SYS_MULTILANGUAGE` 테이블 베트남어 컬럼 확장, `EumLocaleResolver`(세션→헤더→기본값 우선순위 로케일 감지), `EumI18nService`(Cache-Aside 패턴 언어 리소스 로드), `GET /api/common/i18n/messages?lang=vi` 엔드포인트, 프론트엔드 `i18n.js` 다국어 모듈, Noto Sans VI 웹 폰트 CDN 적용을 포함한다.

| 항목 | 내용 |
|------|------|
| **수정 파일** | `SYS_MULTILANGUAGE_VI_DDL.sql` (컬럼 추가), `EUM_USER_PROFILE_LOCALE_DDL.sql` (컬럼 추가), `SYS_MULTILANGUAGE_VI_DATA.sql` (DML — 번역 검수 후 적용), `EumI18nMapper.java/xml` (신규), `EumUserLocaleMapper.java/xml` (신규), `EumI18nController.java` (신규), `EumLocaleResolver.java` (신규), `EumI18nService.java` (신규), `i18n.js` (신규), `font-vi.css` (신규), `messages_vi.properties` (신규) — 총 11개 파일 변경 (2개 기존 DDL 추가, 9개 신규) |
| **DB 반영** | `SYS_MULTILANGUAGE` 테이블에 `MSG_VI VARCHAR2(4000)` 컬럼 추가 (기존 데이터 NULL 허용, 영향 없음). `EUM_USER_PROFILE` 테이블에 `LOCALE VARCHAR2(10) DEFAULT 'ko'` 컬럼 추가 (기존 사용자 ko 자동 적용). 번역 DML은 검수 완료 후 별도 적용. |
| **테스트 통과율** | 자료없음 (단위 테스트 14개 TC 수행 후 결과 갱신 필요. 목표: 핵심 2개 클래스 90% 이상 커버리지. UI 렌더링 전 화면 검증 완료 후 갱신.) |
| **요구사항 충족** | 수용. 요구사항 기준(`REQ-EUM-003` — "베트남어 적용", `REQ-COW-002` — "다국어 지원 한국어·영어·베트남어")으로 개발 계획 / 상세 명세 / 소스 변경 요청서 추적키 정합성 확인 완료. 단, 번역 데이터 원어민 검수 완료 및 현지 폰트 로딩 성능 확인 후 최종 충족 여부 확인. |

### ■ 기술적 변경 상세 (Technical Review)

**소스 코드 정합성 (Code Diff)**

| 구분 | Before | After |
|------|--------|-------|
| `EumLocaleResolver.java` | 존재하지 않음 (Smarts4j 기본 LocaleResolver 사용 — ko 고정 또는 단순 헤더 기반, vi 미지원) | 신규 생성. `LocaleResolver` 구현. ① 세션 `USER_LOCALE` → ② Accept-Language 헤더 → ③ 기본값(`ko`) 3단계 우선순위 로케일 결정. 지원 로케일 `["vi","ko","en"]` 화이트리스트 검증 포함. |
| `EumI18nService.java` | 존재하지 않음 | 신규 생성. Cache-Aside 패턴. Redis `eum:i18n:{lang}` 키 우선 조회 → 캐시 미스 시 DB(`SYS_MULTILANGUAGE`) 조회 → Redis 저장(TTL 무제한). 미지원 언어 코드 요청 시 `UnsupportedLocaleException` 발생. DB 조회 결과 없을 시 빈 맵 Fallback (서비스 중단 방지). |
| `EumI18nController.java` | 존재하지 않음 | 신규 생성. `GET /api/common/i18n/messages?lang=vi` (공개) + `PUT /api/common/user/locale` (인증 필요). `@AuthenticationPrincipal`로 현재 사용자 ID 추출하여 본인 locale만 수정 가능. |
| `EumI18nMapper.xml` | 존재하지 않음 | 신규 생성. `CASE #{lang} WHEN 'vi' THEN MSG_VI WHEN 'en' THEN MSG_EN ELSE MSG_KO END` 동적 언어 컬럼 선택 쿼리. |
| `i18n.js` | 존재하지 않음 (한국어 하드코딩 또는 ko 전용 리소스) | 신규 생성. `load(lang)`, `getLabel(key)`, `changeLocale(lang)` 3개 메서드. 키 미존재 시 키 문자열 그대로 반환(서비스 중단 방지). `localeChanged` CustomEvent 발행으로 UI 전체 텍스트 갱신. |
| `font-vi.css` | 존재하지 않음 (베트남어 성조 문자 지원 불가) | 신규 생성. Noto Sans VI CDN `@import`. 기존 한국어 폰트 스택 유지하며 베트남어 문자 범위에 우선 적용. `text-overflow: ellipsis`, `word-break: break-word` 레이아웃 방어 CSS 포함. |
| `SYS_MULTILANGUAGE` (기존 테이블) | `MSG_VI` 컬럼 없음 | `MSG_VI VARCHAR2(4000)` 컬럼 추가. 기존 데이터 `NULL` 허용. |
| `EUM_USER_PROFILE` (기존 테이블) | `LOCALE` 컬럼 없음 | `LOCALE VARCHAR2(10) DEFAULT 'ko'` 컬럼 추가. 기존 사용자 `ko` 자동 적용. |

**DB 인프라 검증**

| 항목 | 내용 |
|------|------|
| DDL/엔티티 변경 | `SYS_MULTILANGUAGE.MSG_VI` 컬럼 추가 — 기존 레코드 NULL 허용으로 기존 데이터 정합성 영향 없음 확인 필요. `EUM_USER_PROFILE.LOCALE` 컬럼 추가 — 기존 사용자 `DEFAULT 'ko'` 자동 적용 확인 필요. |
| 마이그레이션 검증 | 스테이징 DB DDL 실행 후 기존 사용자 `LOCALE = 'ko'` 자동 적용 확인. 기존 다국어 기능(한국어, 영어) 정상 동작 회귀 테스트 수행 필요. |
| Redis 인프라 | `eum:i18n:vi` 키 기존 Redis 키 충돌 여부 확인. 배포 직후 `DEL eum:i18n:vi` 캐시 무효화 실행 확인 필요. |
| 롤백 스크립트 | `ALTER TABLE SYS_MULTILANGUAGE DROP COLUMN MSG_VI;` / `ALTER TABLE EUM_USER_PROFILE DROP COLUMN LOCALE;` 별도 보관. Redis `DEL eum:i18n:vi` 수동 삭제. |
