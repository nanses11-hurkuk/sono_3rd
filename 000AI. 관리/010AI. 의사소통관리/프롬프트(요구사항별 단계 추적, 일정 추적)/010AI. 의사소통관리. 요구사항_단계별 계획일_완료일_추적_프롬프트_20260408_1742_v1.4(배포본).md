# 프롬프트: 요구사항 ID별 산출물 이행 현황 체크 (계층형 헤더 매트릭스)

## 1. 목적
본 프롬프트는 프로젝트의 모든 요구사항(ID)에 대하여 분석, 설계, 개발, 단위테스트 단계별 계획일과 종료일을 작성되었는지
각 추적서 및 폴더 내 파일을 통해 검증하고, 이를 **단계별 개발 일정 계획 가로 확장형 매트릭스**로 정리하는 데 목적이 있다.

## 2. 조사 대상 경로 및 파일

### [분석 단계] (200AI)
- **계획:** `200AI. 분석\200AI_C_020. 분석 계획(Dev)`
- **결과:** `200AI. 분석\200AI_C_040. 분석 결과(Dev)`

### [설계 단계] (300AI)
- **계획:** `300AI. 설계\300AI_D_020. 설계 계획(Dev)`
- **결과:** `300AI. 설계\300AI_D_040. 설계 결과(Dev)`

### [구현 단계] (400AI)
- **계획:** `400AI. 구현\400AI_E_020. 개발 계획(Dev)`
- **결과:** `400AI. 구현\400AI_E_040. 개발 상세 명세서(Dev)'

### [단위테스트 단계] (400AI)
- **계획:** `400AI. 구현\400AI_E_100. 단위테스트 계획(Dev)`
- **결과:** `400AI. 구현\400AI_E_120. 단위테스트 결과(Dev)`

## 3. 수행 지침
1. **ID 추출:** `231.유티아이베트남 그룹웨어 시스템 구축_요구사항정의 및 GAP분석서_20260407_2_v1.0.xlsx`를 기준으로 전체 ID 리스트를 저장해.
2. **2단 헤더 구성:** 상단에 단계를 표시하고, 하단에 상세 항목을 배치하여 시각적으로 구분하라.
3. 날짜 표시 2개
   3-1  계획일 :  단계별 계획 폴더 요구사향 ID의  계획일(시작/종료일) 또는 작업일, 없으면 표기하지 마
   3-2  종료일 : 단계별 결과 폴더 요구사향 ID의  종료일(시작/종료일)  또는 작업일, 없으면 표기하지 마

4. 요구사항별 상태 표시가 없는 항목은 요구사항 ID만 기입해줘
4. **결과 파일 생성:** 모든 체크가 완료되면 최종 결과물을 **Markdown(.md) 파일**로 생성하라.
   - **저장 경로:** `000AI. 관리\010AI. 의사소통관리\`
   - **파일 이름 형식:** `010AI. 의사소통관리. 요구사항_계획일 실행날짜_추적 매트릭스현황_[년월일시분]_v[일련변호].md` (기존 파일이 존재할 경우 버전을 0.1씩 올린다.)
