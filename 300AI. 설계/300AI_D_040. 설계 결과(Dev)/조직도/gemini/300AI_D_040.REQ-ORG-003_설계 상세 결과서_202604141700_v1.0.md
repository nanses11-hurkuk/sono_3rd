## (2) 개발 최종 설계 내용

### 1. 아키텍처 (Architecture)
- **컴포넌트 구조**: 
    - **UnifiedOrgManager (Service)**: 멀티 테넌시(SaaS) 환경에서 도메인 격리를 우회하여 통합 데이터를 제공하는 서비스.
    - **coviTree (UI Component)**: `Common.js` 기반의 비동기 트리 렌더러 (기존 재활용 및 확장).
    - **GlobalSearchSvc**: 전사 사원 통합 검색 엔진.
- **시퀀스 다이어그램**:
    1. UI(조직도 팝업) -> `OrgApiCon.getUnifiedTreeList.do` 호출.
    2. `UnifiedOrgManager` -> `sys_domain` 목록 기반으로 V1, V2 각각의 최상위 부서(`DeptID`) 조회.
    3. `OrgMapper`에서 도메인 필터(`DomainID = ?`)를 제거한 통합 SQL 실행.
    4. UI(`coviTree`)에서 계열사별 커스텀 아이콘 적용하여 트리 렌더링.

### 2. 데이터 설계 (Data Design)
- **통합 조회 전략**: `sys_object_user`, `sys_object_dept` 테이블에 대해 `domainID`를 무시하고 `CompanyCode`로 그룹화하여 조회.
- **캐시 저장 전략**: Redis Key `ORG_UNIFIED_TREE_ALL`로 통합 트리 구조 캐싱.

### 3. 인터페이스 (API Specification)
- **IF-COV-USR-290 (Extended)**:
    - **Endpoint**: `/covicore/api/org/getUnifiedList.do`
    - **Parameter**: `domainID=ALL`, `searchText=...`
    - **Response**: `CoviList` (통합 부서 및 사원 목록).
