## (2) 개발 최종 설계 내용

### 1. 아키텍처 (Architecture)
- **컴포넌트 구조**: 
    - **CoviJob (Batch Layer)**: 스케줄링 및 배치 실행 로그 관리 (기존 재활용).
    - **HRSyncSvc (Business Layer)**: ERP 데이터 연동 및 유효성 검사 로직 총괄.
    - **MSSQLMapper (Data Access Layer)**: 영림원 ERP(MSSQL) View 전용 Mapper.
- **시퀀스 다이어그램**:
    1. `CoviJob` 트리거 -> `HRSyncSvc.syncV2ERPData()` 호출.
    2. `MSSQLMapper`를 통해 `dbo.GW_User_View` 전체 데이터 페칭.
    3. `HRSyncSvc`에서 사번 중복 및 `sys_base_code`를 통한 직위/직책 코드 변환.
    4. `OrganizationManageMapper`를 호출하여 `sys_object_user`에 `UPSERT` 반영.
    5. `RedisShardsUtil.flushCache()` 호출하여 조직도 캐시 갱신.

### 2. 데이터 설계 (Data Design)
- **스키마/ERP**: 
    - ERP View: `dbo.GW_User_View` (EmpNo, DeptCode, DisplayName, JobLevelCode, IsUse 등).
    - GW Staging: `org_sync_staging` (임시 적재 및 로그 추적용).
- **캐시 전략**: Redis Key `ORG_TREE_{CompanyCode}` 및 `USER_INFO_{UserCode}` 삭제 후 Lazy Loading 유도.

### 3. 인터페이스 (API Specification)
- **IF-COV-USR-702 (Extended)**:
    - **Method**: `POST`
    - **URI**: `/covicore/admin/org/syncERPNow.do`
    - **Summary**: 관리자 페이지에서 V2 계열사 인사 정보 수동 동기화 트리거.

### 4. 구현 가이드 (Pseudo Code/Logic)
```java
// HRSyncSvcImpl.java
@Transactional(value="mssqlTransactionManager")
public void syncERPData() {
    List<HRUserDTO> erpUsers = mssqlMapper.selectERPUserView();
    for(HRUserDTO user : erpUsers) {
        if(validateUser(user)) {
            String gwJobCode = codeSvc.transformCode("JobLevel", user.getErpJobCode());
            orgMapper.upsertUser(user, gwJobCode);
        }
    }
}
```
