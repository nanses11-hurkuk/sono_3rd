## (2) 개발 최종 설계 내용

### [아키텍처 설계]
- **컴포넌트 구조**: Smarts4j `CustomAuthenticator` 인터페이스를 상속받은 `VendorAuthPlugin` 구현. 
- **시퀀스 다이어그램**: 
  1. 외부 사용자 로그인 요청 → 2. `VendorAuthPlugin`에서 외부 DB/API 검증 → 3. 성공 시 WebSocket 세션 토큰 발행 → 4. 채팅 채널 권한 필터 적용.

### [데이터 설계]
- **저장 전략**: 외부 계정 매핑 정보를 위한 `EUM_VENDOR_USER_MAP` 테이블 신설.
- **캐시**: Redis를 활용하여 세션 권한 정보(Session ACL)를 TTL 2시간으로 관리.

### [인터페이스(API Specification)]
- **Endpoint**: `POST /api/eum/auth/external`
- **Request**: `{ "vendorId": "V_001", "extUserId": "user_a", "secret": "..." }`
- **Response**: JWT 기반 Access Token 및 WebSocket Endpoint URL.
