## 1. 양식 스키마 개요

### 1.1 목적

전자결재 시스템에서 사용될 모든 결재양식의 스키마를 정의하여, 양식의 특성, 기능, 연동 정보, 프로세스, 그리고 각 양식별 옵션 설정을 체계적으로 관리합니다.

### 1.2 양식 분류

전자결재 양식은 다음과 같이 분류됩니다:

- **전사공통 양식:** 모든 부서에서 사용 가능한 기본 양식
- **부서별 양식:** 특정 부서(인사, 재경 등)에서 사용하는 양식

### 1.3 양식 구성 요소

각 양식은 다음과 같은 요소로 구성됩니다:

1. **양식 기본정보**
   - 양식명 (Form Name)
   - 양식키 (Form Key / FORM_PREFIX)
   - 양식분류 (서식 위치)
   - 버전 (Version)
   - 정렬순서 (Sort Order)

2. **양식 특성**
   - 개발 여부
   - 사용 여부
   - 작성 권한
   - 설명

3. **프로세스 정보**
   - 프로세스 유형 (보고/품의, 다수수신처, 신청)
   - 결재선 구성

4. **연동 정보**
   - 연동 여부
   - 연동 시스템 (SAP, MSSQL)
   - 연동 시점 (RFC, DB Link)
   - 연동 방향 (Import/Export)

5. **결재 옵션**
   - 합의/협조 옵션
   - 배포 옵션
   - 참조/회람 옵션
   - 기타 기능 옵션

## 3. 양식 스키마 속성

### 3.1 속성 분류

양식의 모든 속성은 다음과 같이 분류됩니다:

#### A. 기본정보 속성
- 번호 (No.)
- 양식분류 (서식위치)
- 양식명 (Form Name)
- 양식키 (FORM_PREFIX)
- 버전 (Version)
- 정렬 (Sort Order)
- 양식 설명 (Description)
- 개발여부 (Development Status)
- 사용여부 (Usage Status)
- 작성 권한 (Authority)

#### B. 프로세스 속성
- 프로세스 유형 (보고/품의, 다수수신처, 신청)
- 연동 여부 (Integration Status)
- 연동 시스템 (SAP, MSSQL)
- 연동 방식 (RFC, DB Link)

#### C. 결재 옵션 속성

**합의 옵션:**
- 개인합의(순차)
- 개인합의(병렬)
- 부서합의(순차)
- 부서합의(병렬)
- (수신부서)부서합의(순차)
- (수신부서)부서합의(병렬)

**협조 옵션:**
- 개인협조(순차)
- 개인협조(병렬)
- 부서협조(순차)
- 부서협조(병렬)

**기타 결재 옵션:**
- 동시결재
- 합의자 수 제한
- 협조부서 삭제
- 강제합의
- 합의단계확인
- 합의부서 내 반려

#### D. 배포/회람 속성
- 배포 사용 여부
- 배포그룹 사용 여부
- 회신 옵션
- 수신자 사용
- 수신결재자 사용
- 수신부서 사용

#### E. 담당자 관련 속성
- 담당업무 사용 (계열사/사업장별)
- 담당부서 사용 (계열사/사업장별)
- 담당수신자 사용

#### F. 감사/준법 속성 (신청 프로세스 제외)
- 개인감사 사용
- 부서감사 사용
- 개인준법 사용
- 부서준법 사용

#### G. 참조/회람 속성
- 수신공람
- 확인결재
- 참조결재
- 개인참조 사용
- 부서참조 사용
- 사전참조 사용

#### H. 저장 위치 속성
- 품의함 저장
- 발신함 저장
- 개인합의 부서함저장
- 수신처리함저장
- 특정권한함

#### I. 결재선 관리 속성
- 결재문서 결재선 출력형태 (Table, List 등)
- 결재선중복체크
- 결재자 수 제한
- 부서장결재단계사용
- 개인협조(순차) 편집

#### J. 편집 권한 속성
- 결재자 내용변경
- 수신부서 내용변경
- 기안자 편집
- 지정반려
- 부서내반송

#### K. 기타 기능 속성
- 담당자버튼사용
- 1인결재사용
- 수신부서 1인결재
- 전달사용
- 결재암호사용
- 서명사용
- 수신처 개별 버튼사용
- 승인/반려버튼 활성화
- 부서합의 접수/반려 버튼 사용
- 결재시Validation체크
- 자동승인 사용
- 검토요청/검토 사용

#### L. 문서 번호 및 분류 속성
- 문서번호 발급 (기본)
- 문서번호선발급
- 기밀문서사용
- 기안취소
- 승인취소
- 대외공문변환

#### M. 결재자/기록 관리 속성
- 현결재자변경
- 의견사용
- EDMS연동(타시스템)
- 기록물철 사용

#### N. 특수 결재 방식 속성
- 보류사용
- 전결사용
- 후결사용
- 대결
- 회람

### 3.2 속성값 정의

#### 기본값
- **Y:** 사용함 / 활성화
- **N:** 사용하지 않음 / 비활성화
- **-:** 해당 없음
- **NaN/공백:** 미정의

#### 프로세스 유형
- **보고/품의:** 기본적인 상하 결재 구조
- **다수수신처:** 하나의 발신부서에서 여러 수신부서로 배포
- **신청:** 신청자가 요청하고 처리부서에서 처리하는 구조

## 5. 양식 샘플 구조

### 5.1 일반양식 샘플 - 출장품의서

**양식 구성:**
1. 설명 영역
2. 상단 공통정보 (문서분류, 보안등급, 보존년한)
3. 참조/회람 정보
4. 제목
5. 본문 영역
6. 결재선 정보

**주요 필드:**
- 작성자 정보 (성명, 부서)
- 문서 메타정보 (문서번호, 분류, 보안등급, 보존년한)
- 의견 작성 가능

---

### 5.2 연동양식 샘플 - 계약품의서

**양식 특징:**
- SAP RFC 연동을 통한 계약 정보 자동 인입
- 정형 양식으로 구성

**주요 필드:**
- 계약명
- 계약기간 (SAP 연동)
- 계약금액 (SAP 연동)
- 지급조건 (SAP 연동)
- 업체정보 (공급업체 코드, 사업자등록번호 등)
- 기타사항

**Interface 정의:**
- RFC 명: ZMM_gcacontObj
- 방향: Import (SAP → 전자결재)
- 데이터 형식:
  - Z_DOC_ID (ERP Key, CHAR 10) - 예)10000004
  - 추가 필드별 매핑 정보

## 6. 스키마 설계 주요 고려사항

### 6.1 연동 전략

- 현재 샘플 양식들은 대부분 연동 없음 상태
- 계약품의서 등 중요 양식은 SAP RFC 연동 계획
- MSSQL DB Link를 통한 추가 시스템 연동 가능

### 6.2 프로세스별 특징

**다수수신처 프로세스:**
- 발신부서에서 여러 수신부서로 문서 배포
- 각 수신부서에서 독립적으로 처리
- 배포 기능과 수신처 관리가 핵심

**보고/품의 프로세스:**
- 기본적인 상하 결재 구조
- 부서장 경유를 통한 의사결정 프로세스

**신청 프로세스:**
