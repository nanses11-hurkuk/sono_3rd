## (2) 개발 설계 수행 내용

### 1. AI 및 개발자 의견 취합
- **AI 의견**: `smarts4j.coviexchange` 모듈에 구현된 `LdapUserSvc`와 `LdapManager`를 활용하여 AD 동기화 로직을 일원화할 것을 권장함. AADC(Azure AD Connect)가 30분 주기로 Local AD 변경사항을 M365로 전파하므로, GW는 Local AD 업데이트에 집중하는 'AD-First' 전략을 채택함.
- **개발자 의견**: AD 계정 생성 시 `userPrincipalName(UPN)` 생성 규칙을 `LogonID@domain.com`으로 표준화하고, 메일그룹 동기화는 `ExchangeRestProxy`를 통해 처리함. 베트남 현지 네트워크 지연을 고려하여 비동기 큐(Queue) 방식의 처리를 검토함.

### 2. 매핑/GAP 분석 및 대응
- **매핑/GAP 분석**: AD 암호 정책과 그룹웨어 암호 정책의 상이함으로 인한 동기화 실패 리스크.
- **GAP 대응 방안**: 암호 동기화 대신 SSO(Single Sign-On) 인증 연동을 통해 AD를 통합 인증 저장소로 활용하며, GW는 계정 상태(활성/비활성) 동기화에 집중함.
- **분류**: 인프라, 인터페이스, 기능
- **수용여부**: 수용

### 3. 상세 설계 내역

#### A. 아키텍처 (Architecture)
- **컴포넌트 구조**: 
    - `ADSyncSvc`: LDAP API를 호출하여 AD 객체를 관리하는 서비스.
    - `LdapManager`: AD 접속 컨텍스트 관리.
    - `ExchangeRestProxy`: `coviexchange` 모듈 연동 프록시.
- **시퀀스 다이어그램**:
    1. 인사 동기화 완료 후 `ADSyncJob` 호출.
    2. `ADSyncSvc`가 변경된 사용자 목록 추출.
    3. `LdapManager`를 통해 AD 내 `User` 객체 생성/수정/삭제.
    4. `ExchangeRestProxy`를 통해 M365 사서함 활성화 요청.

#### B. 데이터 설계 (Data Design)
- **AD Mapping**: `sAMAccountName`, `userPrincipalName`, `displayName`, `mail` 속성 매핑.
- **캐시 전략**: AD 상태 정보를 Redis에 캐싱하여 조회 성능 최적화.

#### C. 인터페이스 (API Specification)
- **IF-EXCH-006**: `v1/org/user` (AD 사용자 관리 API) 재활용.
- **LDAP**: Port 636 (LDAPS) 통신 권장.

#### D. 구현 가이드 (Pseudo Code)
```java
// LdapUserSvcImpl.java 재활용
public void syncToAD(UserDTO user) {
    LdapContext ctx = ldapManager.getContext();
    ModificationItem[] mods = new ModificationItem[3];
    mods[0] = new ModificationItem(DirContext.REPLACE_ATTRIBUTE, new BasicAttribute("displayName", user.getDisplayName()));
    mods[1] = new ModificationItem(DirContext.REPLACE_ATTRIBUTE, new BasicAttribute("mail", user.getMailAddress()));
    mods[2] = new ModificationItem(DirContext.REPLACE_ATTRIBUTE, new BasicAttribute("department", user.getDeptName()));
    
    ctx.modifyAttributes(user.getDn(), mods);
}
```

#### E. 패키지 및 클래스 설계
- **패키지**: `egovframework.coviexchange.api.v1.org.service.impl`
- **DTO**: `ADUserDTO`, `MSEWSUserDTO`

#### F. 외부 연동
- **대상 시스템**: Local Active Directory, Microsoft 365
- **연동 방식**: LDAP (JNDI), Exchange REST API
