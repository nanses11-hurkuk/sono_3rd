## (2) 개발 설계 수행 내용

### 1. AI 및 개발자 의견 취합
- **AI 의견**: `smarts4j.covicore`의 표준 인사 동기화 인터페이스인 `IF-COV-USR-702`를 기반으로 영림원 ERP와의 연동을 자동화할 것을 제안함. JDBC Direct Connection 방식을 사용하여 중간 파일 없이 DB-to-DB로 데이터를 취합하여 정합성을 극대화함.
- **개발자 의견**: `HRConnectCode.java`에서 검증된 MSSQL 접속 로직을 `SyncSvc`에 통합하고, 배치 스케줄링은 기존 `CoviJob` 프레임워크를 활용하여 매일 01:00에 실행되도록 구성함. V1/V2 계열사 구분은 `DomainID` 또는 별도 연동 플래그를 통해 처리함.

### 2. 매핑/GAP 분석 및 대응
- **매핑/GAP 분석**: ERP View와 그룹웨어 테이블 간의 직위/직책 코드 체계 상이. 영림원 ERP는 정수형 코드를 사용하나 그룹웨어는 유니크한 문자열 키를 요구함.
- **GAP 대응 방안**: 동기화 쿼리 내에서 `CASE WHEN` 또는 별도 매핑 테이블(`sys_base_code` 활용)을 통해 코드를 변환 적재함.
- **분류**: 기능, 인터페이스
- **수용여부**: 수용

### 3. 상세 설계 내역

#### A. 아키텍처 (Architecture)
- **컴포넌트 구조**: 
    - `CoviJobScheduler`: Quartz 기반 배치 트리거.
    - `HRConnector`: MSSQL 연동 및 데이터 추출 엔진.
    - `SyncStoredProcedure`: 추출된 데이터를 GW 본 테이블로 반영하는 SP.
- **시퀀스 다이어그램**:
    1. `CoviJob` 트리거 실행.
    2. `HRConnector`가 ERP View(`dbo.GW_User`) 데이터 조회.
    3. GW Staging 테이블(`org_hr_sync_staging`)에 `Bulk Insert`.
    4. 코드 변환 및 유효성 검사 수행.
    5. 본 테이블(`sys_object_user`, `sys_object_dept`)에 `Merge(Upsert)` 수행.

#### B. 데이터 설계 (Data Design)
- **ERP 연계 Schema**: 영림원 ERP View(`dbo.GW_User`, `dbo.GW_Department`) 매핑.
- **저장 전략**: Staging 테이블을 거쳐 최종 반영함으로써 동기화 오류 시 롤백 및 로그 추적 용이성 확보.

#### C. 인터페이스 (API Specification)
- **IF-COV-USR-702**: 표준 인사 동기화 인터페이스 확장.
- **DB Connection**: `mssql-jdbc` 드라이버 활용, Port 1433 개방 필요.

#### D. 구현 가이드 (Pseudo Code)
```java
// HRConnectCode.java 로직 재활용
public void executeSync() {
    Connection erpConn = DriverManager.getConnection(DB_URL, DB_USER, DB_PASS);
    List<Map> erpData = fetchRows(erpConn, "SELECT * FROM dbo.GW_User");
    
    for(Map row : erpData) {
        // 코드 매핑 및 유효성 검사
        String jobLevelCode = mappingSvc.getGWCode("JobLevel", row.get("JobLevelCode"));
        // DB Upsert
        orgUserMapper.upsertUser(row, jobLevelCode);
    }
}
```
