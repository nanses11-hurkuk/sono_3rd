## (2) 개발 설계 수행 내용

### AI 의견

- **coviexchange 모듈에 LDAP/AD 연동 이미 구현됨:** `smarts4j.coviexchange`는 EWS(Exchange Web Services)뿐 아니라 PowerShell 및 **LDAP 연동**을 통한 AD 인프라 제어 기능을 포함합니다. `RestGroupCon`(`v1/org/group`), `RestUserCon`(`v1/org/user`) 컨트롤러가 기존 구현되어 있어 AD 그룹·사용자 관리에 재활용 가능합니다.
- **coviexchange 멀티 테넌시 구조 확인:** 모든 요청은 HTTP Header의 `domainId`를 기반으로 테넌트별 Exchange/LDAP 설정(`EWSConfigDTO`)을 로드합니다. V1·V2 통합 AD 구조에서도 `domainId`로 계열사를 구분할 수 있습니다.
- **M365 Graph API 연동 기존 구현 확인:** `covicore`의 `M365GraphAPICon`에 `setOAuthToken`, `setM365Mapping`, `setGroupMember`, `setPresenceForGW` 등 M365 연동 기능이 이미 구현되어 있습니다. M365 그룹 멤버 동기화(`setGroupMember`)를 GW→AD→AADC 흐름의 보완 수단으로 활용 가능합니다.
- **AD_IsUse 컬럼 활용:** `orgs_orgperson.AD_IsUse` 컬럼이 이미 GW 스키마에 존재합니다. REQ-ORG-001 동기화 완료 후 `AD_IsUse=Y`인 사용자를 대상으로 coviexchange의 AD 동기화를 트리거하는 구조로 연계합니다.
- **coviexchange 인증 방식:** PowerShell 연동 시 Kerberos 인증 사용. LDAP 연동으로 AD 사용자 속성 실시간 동기화. V1·V2 단일 AD 통합 후 `domainId` 기반으로 처리합니다.
- **개발 테넌트 미확보:** M365GraphAPICon 기존 구현이 있으나, 개발 테넌트 없으므로 실 연동 테스트 불가. `setGroupMember` 등 Graph API 호출부는 Mock으로 선 개발 후 테넌트 확보 시 전환합니다.
- **GW→AD 동기화 트리거:** `OrgSyncCon.synchronize` 완료 후 `JobExecutionListener` 또는 Spring `ApplicationEvent`로 coviexchange `RestUserCon` / `RestGroupCon` API를 호출합니다.

   HTTP POST → covicore m365/setGroupMember.do

6. M365 상태 정보 동기화 (기존 기능)
