### 2-1. AI 의견

두 소스코드(`covimail.zip`, `smarts4j_coviexchange.zip`)를 직접 분석한 결과, 현재 구현 상태와 GAP 해소 방향에 대해 다음과 같이 의견을 제시한다.

**[재활용 가능 기존 구현]**

| 구성요소 | 위치 | 재활용 가능 내용 |
|---------|------|---------------|
| `EWSCommon.getExchangeService()` | `smarts4j_coviexchange/EWSCommon.java:99~165` | `ExchangeService` 생성, EWS 버전 분기, Impersonation 설정, **타임아웃 30초 이미 적용** (`exchangeService.setTimeout(30000)`) |
| `ResourceManager` | `smarts4j_coviexchange/ResourceManager.java` | `ConcurrentHashMap` 기반 도메인별 설정 캐시 — 재연결 로직 추가 대상 |
| `PowerShellCommand.setSession()` | `smarts4j_coviexchange/PowerShellCommand.java:48` | Exchange PowerShell 세션 생성 구현 완료 — 재연결 탐지 로직 추가 필요 |
| `ExchangeConfigDTO` | `smarts4j_coviexchange/ExchangeConfigDTO.java` | EWS/AD/PowerShell 설정 DTO 완비 |
| `MailSvc` 인터페이스 | `smarts4j_coviexchange/MailSvc.java` | 메일 CRUD, 폴더, 구독, 규칙, 카테고리 전체 메서드 정의 완료 |
| `CoviMSAPIUtil` | `covimail/CoviMSAPIUtil.java` | Graph/Exchange 모듈 분기 처리, ServiceContext 구성 기반 존재 |
| `BigAttachHelper` | `covimail/BigAttachHelper.java` | 대용량 첨부 기반 클래스 존재 — 업로드 세션 확장 대상 |

**[핵심 GAP 분석 — 소스코드 직접 확인 결과]**

- **GAP-WML-001-01 (타임아웃)**: `EWSCommon.java:158`에 `exchangeService.setTimeout(30000)` 이미 적용됨. covimail 측 IMAP 세션 타임아웃은 `mail.properties`에 별도 설정 필요
- **GAP-WML-001-02 (AccessToken 갱신)**: `EWSCommon.java:134~136`에 토큰 인증 코드가 주석 처리(`// service.setHttpHeaders("Authorization", "Bearer " + accessToken)`)됨 — MSAL4J 기반 활성화 필요
- **GAP-WML-001-07 (환경변수 분리)**: `ResourceManager`가 `domainId` 키로 `ConcurrentHashMap` 관리 중 — 구조는 갖춰져 있으나 Redis 기반 동기화 미적용
- **GAP-WML-001-09 (웹훅 구독 갱신)**: `MailSvc`에 `createSubscription()` / `unSubscription()` 메서드 정의되어 있으나 자동 갱신 스케줄러 없음
- **PowerShell 세션**: `setSession.ps1`이 `-Authentication Kerberos`로 고정되어 있어 OAuth2 전환 시 스크립트 수정 필요

```java
// [신규] TokenRefreshService.java
// 위치: egovframework.covimail.common.service.impl.TokenRefreshService
public class TokenRefreshService {
    // RedisDataUtil 기 사용 환경 재활용
    public String getOrRefreshToken(String domainId) throws Exception {
        // 1. Redis에서 캐시 토큰 조회
        String token = RedisDataUtil.get("msAccessToken:" + domainId);
        if (token != null && !token.isEmpty()) return token;
        
        // 2. 만료 or 없음 → MSAL4J 재발급
        String clientId = RedisDataUtil.getBaseConfig("OauthClientId", domainId);
        String clientSecret = RedisDataUtil.getBaseConfig("OauthClientSecret", domainId);
        String tenantId = RedisDataUtil.getBaseConfig("OauthTenantId", domainId);
        
        ConfidentialClientApplication app = ConfidentialClientApplication.builder(
            clientId,
            ClientCredentialFactory.createFromSecret(clientSecret))
            .authority("https://login.microsoftonline.com/" + tenantId)
            .build();
        
        ClientCredentialParameters params = ClientCredentialParameters
            .builder(Collections.singleton("https://outlook.office365.com/.default"))
            .build();
        
        IAuthenticationResult result = app.acquireToken(params).get();
        token = result.accessToken();
        
        // 3. Redis 캐시 저장 (TTL: 3500초)
        RedisDataUtil.setEx("msAccessToken:" + domainId, token, 3500);
        return token;
    }
}
```

#### GAP-WML-001-09 — 웹훅 구독 자동 갱신 스케줄러

```java
// [신규] SubscriptionRenewalScheduler.java
// 위치: egovframework.covimail.common.util.SubscriptionRenewalScheduler
// AsyncTaskAutoDelete 패턴 참조
@Component
public class SubscriptionRenewalScheduler {
    @Autowired private MailSvc mailSvc;       // coviexchange MailSvc 재활용
    @Autowired private RedisDataUtil redis;

    @Scheduled(fixedDelay = 3600000) // 60분 주기
    public void renewSubscriptions() {
        // Redis에서 만료 60분 이내 구독 목록 조회
        Set<String> keys = redis.keys("ewsSubscription:*");
        for (String key : keys) {
            SubscriptionInfo info = redis.getObject(key, SubscriptionInfo.class);
            long minutesLeft = ChronoUnit.MINUTES.between(
                LocalDateTime.now(), info.getExpiresAt());
            if (minutesLeft <= 60) {
                ExchangeService service = EWSCommon.getExchangeService(
                    resourceManager.get(info.getDomainId()).getEWSConfigDTO(),
                    info.getMailAddress());
                // 기존 MailSvc.createSubscription() 재활용
                GWSubscribeEwsDTO dto = new GWSubscribeEwsDTO();
                dto.setSubscriptionId(info.getSubscriptionId());
                mailSvc.createSubscription(service, dto);
                // Redis TTL 갱신
                redis.setEx(key, info, 4200 * 60); // 최대 4200분
            }
        }
    }
}
```
