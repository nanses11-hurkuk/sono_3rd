### 2-1. AI 의견

**[요구사항 해석]**

이음톡 메신저 UI 전반에 베트남어(Tiếng Việt) 언어팩을 추가하는 요구사항이다. 단순 텍스트 번역 제공이 아닌, 다국어 지원(i18n, Internationalization) 아키텍처를 갖추어 베트남어를 포함하고 향후 타 언어도 추가 가능한 구조로 설계해야 한다.

**[기술적 타당성 검토]**

| 검토 항목 | 기술적 판단 | 근거 |
|---------|-----------|------|
| 프론트엔드 i18n 적용 | ✅ 가능 | React/Vue/Angular 등 주요 프레임워크는 `i18next`, `vue-i18n`, `Angular i18n` 등 성숙한 라이브러리 보유 |
| 모바일 앱 i18n 적용 | ✅ 가능 | React Native: `i18next`, iOS: `Localizable.strings`, Android: `strings.xml` 표준 방식 |
| 백엔드 메시지 i18n | ✅ 가능 | Spring `MessageSource` + `messages_vi.properties` 파일 추가로 서버 오류 메시지도 다국어 처리 가능 |
| 베트남어 인코딩 | ✅ 문제없음 | UTF-8 기반 시스템이면 베트남어(유니코드 포함 특수 성조 문자) 처리 가능 |
| DB 저장 | ✅ 문제없음 | MySQL/PostgreSQL UTF8MB4 설정 시 베트남어 저장 완전 지원 |
| 날짜/시간 형식 로컬화 | ✅ 가능 | `Locale.forLanguageTag("vi-VN")` 기반 날짜 포맷 적용 |
| 숫자/통화 형식 로컬화 | ✅ 가능 | 베트남 동(₫, VND) 형식 `NumberFormat` 적용 가능 |
| RTL 레이아웃 | ✅ 불필요 | 베트남어는 LTR(왼쪽→오른쪽) 방향 문자 — 레이아웃 변경 불필요 |
| 번역 품질 관리 | ⚠️ 비용 검토 | 전문 번역사 또는 Native Speaker 검수 필요. AI 자동 번역만으로는 품질 불충분 |

**[핵심 설계 포인트]**

1. **언어팩 파일 구조**: `vi.json` (베트남어), `ko.json` (한국어), `en.json` (영어) 분리 관리
2. **언어 감지 우선순위**: 사용자 설정 → 브라우저 언어 → 시스템 기본값(한국어)
3. **동적 로딩**: 언어팩 파일을 앱 번들에 포함하거나 CDN에서 지연 로딩
4. **날짜/시간 표시**: 베트남 표준시(UTC+7) 반영 — `Asia/Ho_Chi_Minh` TimeZone
5. **서버 오류 메시지**: Spring MessageSource `messages_vi.properties` 파일로 서버 메시지도 베트남어 제공
6. **푸시 알림 메시지**: 모바일 푸시 알림 내용도 사용자 언어 설정에 따라 베트남어 발송

// [프론트엔드] i18next 초기화 (React 기준)
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import LanguageDetector from 'i18next-browser-languagedetector';

i18n
  .use(LanguageDetector)   // 브라우저 언어 자동 감지
  .use(initReactI18next)
  .init({
    resources: {
      ko: { translation: require('./locales/ko/translation.json') },
      vi: { translation: require('./locales/vi/translation.json') },
      en: { translation: require('./locales/en/translation.json') }
    },
    fallbackLng: 'ko',     // 미지원 언어 시 한국어 폴백
    detection: {
      order: ['localStorage', 'navigator'],  // 사용자 설정 → 브라우저 언어
      caches: ['localStorage']
    }
  });

// [프론트엔드] 날짜 형식 로컬화
const formatDate = (dateStr, languageCode) => {
  const date = new Date(dateStr);
  const locale = languageCode === 'vi' ? 'vi-VN' : 'ko-KR';
  return new Intl.DateTimeFormat(locale, {
    year: 'numeric', month: '2-digit', day: '2-digit',
    hour: '2-digit', minute: '2-digit'
  }).format(date);
  // ko: 2026. 04. 15. 오전 10:30
  // vi: 15/04/2026 10:30
};

// [백엔드] Spring LocaleResolver 설정
@Bean
public LocaleResolver localeResolver() {
    AcceptHeaderLocaleResolver resolver = new AcceptHeaderLocaleResolver();
    resolver.setDefaultLocale(new Locale("ko"));
    resolver.setSupportedLocales(Arrays.asList(
        new Locale("ko"), new Locale("vi"), new Locale("en")
    ));
    return resolver;
}

// [백엔드] MessageSource 설정
@Bean
public MessageSource messageSource() {
    ReloadableResourceBundleMessageSource ms = new ReloadableResourceBundleMessageSource();
    ms.setBasenames("classpath:messages/messages");
    ms.setDefaultEncoding("UTF-8");
    ms.setFallbackToSystemLocale(false);
    return ms;
}
// → messages_ko.properties, messages_vi.properties 자동 참조

// [백엔드] 푸시 알림 다국어 발송
public void sendPushNotification(String recipientUserId, String messageKey, Object... args) {
    // 1. 수신자 언어 설정 조회 (Redis 캐시 우선)
    String lang = userLocaleCache.getLanguage(recipientUserId);  // "vi"

    // 2. 해당 언어로 메시지 생성
    Locale locale = new Locale(lang);
    String pushMessage = messageSource.getMessage(messageKey, args, locale);
    // messageKey: "push.newMessage" → "Bạn có tin nhắn mới từ {0}"

    // 3. FCM/APNs 발송 (기존 푸시 서비스 재활용)
    pushService.send(recipientUserId, pushMessage);
}
