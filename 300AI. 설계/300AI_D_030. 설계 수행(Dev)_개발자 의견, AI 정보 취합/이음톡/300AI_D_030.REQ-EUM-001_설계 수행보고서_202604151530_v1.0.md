### 2-1. AI 의견

**[요구사항 해석]**

"vendor 사에서도 사용할 수 있도록"이라는 요구사항은 이음톡 메신저를 내부 임직원(사내)뿐 아니라 **외부 협력업체(Vendor) 사용자도 접속·사용할 수 있도록 확장**하는 것을 의미한다. 이를 위해서는 인증 방식, 사용자 관리, 보안 격리, 접근 제어 등 여러 기술 영역에 걸친 설계가 필요하다.

**[기술적 타당성 검토]**

| 검토 항목 | 기술적 판단 | 근거 |
|---------|-----------|------|
| 외부 사용자 계정 분리 | ✅ 가능 | 내부 AD/LDAP 계정 외 외부 계정 저장소(별도 DB 또는 Guest 계정) 운용 패턴 일반화됨 |
| 외부망 접속 경로 확보 | ⚠️ 인프라 검토 필요 | 방화벽, DMZ 구성, 리버스 프록시(Nginx/Apache) 설정 필요. 베트남 현지 네트워크 정책 확인 요망 |
| 인증 방식 (외부 사용자) | ✅ 가능 | ID/PW 기반 자체 인증 또는 OAuth2(Google, Microsoft) 소셜 인증 적용 가능 |
| 채팅 데이터 보안 격리 | ✅ 가능 | 외부 사용자와 내부 사용자 간 채팅방 분리(채팅방 타입 구분), 내부 전용 채팅방 접근 차단 |
| 기능 제한 (외부 사용자) | ✅ 권장 | 파일 공유 용량 제한, 채팅방 생성 제한, 조직도 접근 차단 등 권한 분리 설계 |
| 세션/토큰 관리 | ✅ 가능 | JWT 기반 액세스 토큰 + 리프레시 토큰 패턴으로 외부 사용자 세션 관리 가능 |

**[핵심 설계 포인트]**

1. **사용자 타입 구분**: `UserType = INTERNAL(사내) / EXTERNAL(외부 vendor)` 컬럼 추가
2. **채팅방 접근 제어**: 채팅방 생성 시 `roomType = INTERNAL_ONLY / EXTERNAL_ALLOWED` 구분
3. **외부 사용자 초대 Flow**: 내부 사용자가 외부 vendor를 이메일로 초대 → 임시 계정 발급 → 초대된 채팅방만 접근
4. **보안**: 외부 사용자 세션 TTL 단축, IP 화이트리스트(선택), 첨부파일 확장자 제한

### 2-9. 구현 가이드 (Pseudo Code / Logic)

> 문서 ID: 300AI_D_030.REQ-EUM-001 | 설계 계획 ID: 300AI_D_020.REQ-EUM-001

```java
// [신규] ExternalAuthFilter.java
// 요청 URL이 /api/external/** 이면 외부 JWT 검증
// 그 외는 기존 내부 인증 필터(LDAP/SSO)로 위임

// [신규] InviteService.createExternalInvite()
public InviteResult createExternalInvite(String roomId, String inviteeEmail,
                                          String inviterId) {
    // 1. 채팅방 room_type 검증 (EXTERNAL_ALLOWED 여부)
    ChatRoom room = chatRoomRepo.findById(roomId);
    if (!room.isExternalAllowed())
        throw new AccessDeniedException("외부 사용자 참여 불가 채팅방");
    // 2. 초대 코드 생성 (UUID, TTL 48h)
    String code = UUID.randomUUID().toString();
    inviteRepo.save(new ChatInviteCode(code, roomId, inviterId, inviteeEmail));
    redisUtil.setEx("ext:invite:" + code, "valid", 172800); // 48h
    // 3. 초대 메일 발송
    mailService.sendInviteEmail(inviteeEmail, code, room.getName());
    return new InviteResult(code, ...);
}

// [신규] ExternalUserService.acceptInvite()
public AuthToken acceptInvite(String code, String displayName,
                               String companyName, String password) {
    // 1. 초대 코드 유효성 검증 (Redis + DB)
    if (!redisUtil.exists("ext:invite:" + code)) throw ...;
    ChatInviteCode invite = inviteRepo.findByCode(code);
    // 2. 외부 사용자 계정 생성 (external_user 테이블)
    ExternalUser user = new ExternalUser(invite.getEmail(), displayName,
                                         companyName, bcrypt(password));
    externalUserRepo.save(user);
    // 3. 채팅방 멤버 등록
    chatRoomMemberRepo.save(new ChatRoomMember(invite.getRoomId(), user.getId()));
    // 4. JWT 발급 (userType=EXTERNAL 클레임 포함)
    String accessToken = jwtUtil.generateToken(user.getId(), "EXTERNAL");
    // 5. 초대 코드 USED 처리
    invite.setStatus("USED");
    return new AuthToken(accessToken, refreshToken);
}
```
