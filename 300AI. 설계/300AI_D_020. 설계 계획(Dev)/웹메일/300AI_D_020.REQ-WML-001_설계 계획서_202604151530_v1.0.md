### 요구사항 (개요)

유티아이 베트남 그룹웨어 시스템 구축 프로젝트에서 기존 covimail(IMAP 기반 웹메일) 솔루션과 Microsoft Exchange/Microsoft 365(Graph API, EWS)를 연동하는 과정에서 식별된 총 **10건의 GAP(GAP-WML-001-01 ~ GAP-WML-001-10)** 을 해소하기 위해 상세 설계를 수행한다.
해당 GAP 항목들은 기능(비즈니스 로직), 비기능(성능/안정성), 인프라(설정/환경) 영역에 걸쳐 있으며, 분석 완료 단계에서 모두 **"수용"** 판정을 받았다.

---

### 요구사항 상세 설명

#### 1. 현황 및 배경

기존 covimail 솔루션은 IMAP/SMTP 기반의 자체 메일 서버(`egovframework.covimail.user.service.impl.MailSvcImpl`)와 MS Exchange 연동 레이어(`MsMailSvcImpl`, `CoviMSAPIUtil`, `CoviMSAPIMailUtil`)로 이원화된 구조를 가진다. 베트남 프로젝트에서는 Microsoft Exchange/M365 기반의 메일 환경을 사용하므로, coviexchange 연동 레이어(`egovframework.msapi`)를 통한 Graph API 및 EWS(Exchange Web Services) 연동이 핵심이다.

소스코드 분석 결과, 현재 연동 구조는 다음과 같다:

```
[사용자 브라우저]
    ↓ HTTP(S)
[MailConAjax.java / MailSettingsCon.java]  ← Spring MVC Controller
    ↓
[MsMailSvcImpl.java]                        ← MS Exchange 연동 서비스 구현체
    ↓
[CoviMSAPIUtil.java / CoviMSAPIMailUtil.java] ← MS API 유틸리티 (Graph/Exchange 컨텍스트 설정)
    ↓
[egovframework.msapi.api.CoviMSAPI]         ← MS Graph API / EWS 호출 레이어
    ↓ REST / SOAP
[Microsoft 365 / Exchange Server]
```

#### 3. 아키텍처 설계 방향 (개요)

```
[설계 원칙]
1. 코어(covimail 기본 패키지) 수정 최소화
   → Smarts4j Extension Point (SvcImplFactory) 활용
   → Decorator 패턴으로 기존 SvcImpl을 감싸는 방식

2. MS Graph API / EWS 연동 레이어 확장
   → CoviMSAPIUtil, CoviMSAPIMailUtil 유틸 클래스 내 신규 메서드 추가
   → egovframework.msapi 패키지 인터페이스를 통한 호출

3. 토큰 관리
   → MSAL4J ConfidentialClientApplication 기반 토큰 캐싱
   → Redis를 토큰 저장소로 활용 (키: "msAccessToken:{DomainId}:{userId}")
   → 토큰 만료 전 10분 갱신 트리거

4. 타임아웃/성능
   → 모든 외부 API 호출에 connectionTimeout(5초), readTimeout(30초) 적용
   → Graph API 페이지네이션 파라미터 표준화
```

### 2-5. 설계 범위 지정

#### 포함 범위 (In-Scope)

1. **기능 설계**
   - AccessToken 갱신 모듈 설계 (MSAL4J 기반)
   - 반복 일정 수정/삭제 API 파라미터 분기 처리 설계
   - 공유 메일함 폴더/메일 조회 API 분기 설계
   - 대용량 첨부파일 업로드 세션 처리 설계
   - 발신 제한 정책 Exchange 연동 설계
   - 웹훅 구독 자동 갱신 스케줄러 설계

2. **비기능 설계**
   - 전체 외부 API 호출 타임아웃 설정 표준화
   - 메일 목록 서버사이드 페이지네이션 설계
   - 토큰 캐시(Redis) 키 구조 및 TTL 설계

3. **인프라/환경 설계**
   - Redis 키 네임스페이스 (`도메인 ID` 기반) 분리 설계
   - Spring Profile 기반 개발/운영 환경 설정 분리
   - `mail.properties` 타임아웃 파라미터 추가

4. **아키텍처 설계**
   - Smarts4j Extension Point (SvcImplFactory 기반 Strategy 패턴) 적용 설계
   - 코어 수정 최소화를 위한 Decorator 패턴 적용 범위 정의

#### 제외 범위 (Out-of-Scope)

- covimail 자체 IMAP 기반 메일 기능 변경 (기존 `MailSvcImpl` 영역)
- 프론트엔드(JSP/JavaScript) 화면 변경 — 백엔드 API 스펙 변경으로 인한 최소한의 수정은 포함하되, UI/UX 재설계는 제외
- Microsoft Azure AD 앱 등록 및 권한 설정 (인프라 팀 담당)
- 데이터 마이그레이션 (별도 이행 계획 수립)
