## (2) 개발 설계 수행 내용

### 1. AI 및 개발자 의견
- **AI 의견**: 외부 협력사(Vendor) 사용자의 접근을 위해 기존 사내 인증 체계와 분리된 외부 전용 인증/인가 프로세스가 필요합니다. 데이터 보안을 위해 외부 사용자는 허용된 협업 채널 및 특정 직원 정보만 조회 가능하도록 강력한 ACL(Access Control List) 적용이 필수적입니다.
- **개발자 의견**: 기존 이음톡의 User 엔티티를 확장하여 사용자 유형(Internal/External) 구분을 추가하고, Vendor용 API Gateway 엔드포인트를 별도로 구성하여 보안성을 확보하겠습니다.

### 2. 매핑/GAP 분석 및 대응 방안
- **매핑/GAP 분석**: 현재 시스템은 사내 AD(Active Directory) 연동 기반으로 외부 계정 생성 및 관리 로직이 부재함.
- **GAP 대응 방안**: 외부 파트너 관리용 DB 스키마를 추가하고, 파트너 전용 로그인 인터페이스 및 JWT 기반 토큰 발급 로직을 신설함.
- **분류**: 기능, 인터페이스
- **수용여부**: 수용
